# 참가자 일일 회고

> 해커톤 종료 후 짧은 회고를 통해 배움을 정리합니다

* * *

## 🎯 한 줄 정의

**회고 = 오늘 경험한 것을 구조화하여 다음 행동으로 연결하는 과정**

* * *

## 왜 필요한가?

해커톤이 끝나면 에너지와 함께 기억도 빠르게 사라집니다. **5분만 투자**해서 정리하면 오늘의 경험이 실무에 남습니다.

* * *

## 📋 회고 양식

### 1. 오늘 만든 것

```
앱 이름: ___________________
배포 URL: ___________________
한 줄 설명: ___________________
```

### 2. 잘 된 것 (Keep)

```
- ___________________
- ___________________
```

### 3. 어려웠던 것 (Problem)

```
- ___________________
- ___________________
```

### 4. 다음에 시도할 것 (Try)

```
- ___________________
- ___________________
```

### 5. 한 줄 소감

```
"___________________"
```

* * *

## 💡 회고 팁

- **솔직하게**: 잘 안 된 것도 적으면 다음에 피할 수 있습니다
- **구체적으로**: "어려웠다" → "프롬프트에서 입출력을 명확히 안 적어서 3번 재시도했다"
- **행동으로**: "다음엔 잘 하겠다" → "다음엔 프롬프트에 입력/출력 예시를 꼭 포함하겠다"

* * *
