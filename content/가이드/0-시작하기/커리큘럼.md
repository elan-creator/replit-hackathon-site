# Replit 해커톤 커리큘럼

> 4시간 30분 안에 아이디어를 실행 가능한 앱으로 만드는 해커톤

* * *

## 🎯 개요

| 항목 | 내용 |
| --- | --- |
| **시간** | 4시간 30분 (09:30~14:00) |
| **대상** | 비개발자 포함 현업팀 (PM, 디자이너, 운영, 기획, 마케팅, 리더) |
| **목표** | 참가자 전원이 **실행 가능한 MVP 링크**를 확보 |
| **도구** | Replit (브라우저만 있으면 OK) |

* * *

## 💡 이 해커톤이 특별한 이유

| 기존 해커톤 😰 | 이 해커톤 ✨ |
| --- | --- |
| 8시간 이상 소요 | **4시간 30분 압축형** |
| 개인 Pro 계정 선결제 필요 | **실습 토큰 비용 운영 측 지원** |
| 로컬 개발환경 셋업 필요 | **브라우저 + Replit 계정으로 즉시 시작** |
| 배포까지 복잡한 과정 | **클릭 한 번으로 배포 자동화** |
| 결과물 없이 끝나는 경우 多 | **전원 MVP 링크 확보가 목표** |

> 💡 **"코딩을 전혀 몰라도 참가할 수 있나요?"** — 네! 여러분의 업무 지식이 곧 최고의 프롬프트입니다. Replit Agent가 코드를 만들고, 여러분은 **무엇을 만들지**에 집중하면 됩니다.

## 📌 파트별 핵심 포인트

### Part 0: OT + 팀빌딩 + 강의 — "준비하기"

- 해커톤의 목표와 마인드셋 공유
- 팀 구성 및 아이디어 피칭
- Replit으로 바이브 코딩하는 방법 강의

### Part 1: MVP 개발 + 개밥먹기 — "일단 돌아가게"

- Replit Agent에게 프롬프트 → **동작하는 앱 1개** 확보
- 만든 MVP를 직접 테스트하고 문제점 발견
- **핵심**: 완성도보다 속도. 가장 중요한 기능 1개에 집중

### Part 2: 추가개발 + 피드백 + 최종개선 — "실제로 쓸 수 있게"

- 기능 고도화, UI 개선 (점심 먹으면서 진행)
- 다른 사람에게 **설명 없이** 써보게 하고 피드백 수집
- 피드백 반영하여 최종 개선 및 제출

### Part 3: 소감공유 및 마무리 — "가져갈 것 확인"

- 소감 나누기
- 향후 활용 계획 수립
