# Replit 소개

> Replit = 브라우저에서 코드 작성, 실행, 배포까지 한 번에 되는 올인원 개발 플랫폼

* * *

## 🎯 한 줄 정의

**Replit은 복잡한 설치 없이 브라우저만 있으면 앱을 만들고 배포할 수 있는 클라우드 기반 개발 환경입니다.**

* * *

## 왜 Replit인가? — 개발 장벽을 허물다

### 전통적 개발 과정 (비개발자에게 너무 어려움)

```
코드 에디터 설치 → 프로그래밍 언어 설치 → 패키지 매니저 설치 
→ 프레임워크 설치 → 환경변수 설정 → 로컬 서버 실행 → 코드 작성 → 빌드 → 서버 설정 → 배포
```

**문제점**: 초보자 기준 반나절~하루가 걸리고, 대부분의 아이디어가 "환경 설정" 단계에서 좌절됩니다.

### Replit 개발 과정 (누구나 5분 안에 시작)

```
replit.com 접속 → 로그인 → Create Repl 클릭 → "이런 앱 만들어줘" (Agent 모드) → Deploy 클릭
```

**소요 시간**: 5분 이내.

* * *

## 📋 해커톤에서 알아야 할 핵심 기능 3가지

### 1. Replit Agent (AI 기반 앱 생성)

AI가 자연어 설명만으로 전체 앱을 만들어줍니다.

- **자연어로 요청**: "할 일 관리 웹앱 만들어줘"
- **전체 앱 코드 생성**: 파일 구조, 의존성, 기본 코드까지 자동 처리
- **자동 디버깅**: 에러 발생 시 자동으로 수정 시도

> 💡 **비개발자는 "무엇을" 만들지에 집중하고, "어떻게" 만드는지는 Agent에게 맡기면 됩니다.**

### 2. 즉시 실행 (Instant Run)

코드를 작성하면 **별도의 빌드 과정 없이 즉시 결과물을 확인**할 수 있습니다.

- **실시간 미리보기**: 웹앱이 바로 동작하는 모습을 확인
- **빠른 피드백 루프**: 수정 즉시 결과 확인
- **에러 즉시 감지**: 문제 발생 시 바로 에러 메시지 표시

### 3. 원클릭 배포 (Deploy)

만든 앱을 인터넷에 공개하여 누구나 URL로 접속할 수 있게 합니다.

| 배포 유형 | 설명 | 적합한 경우 |
| --- | --- | --- |
| Static | 정적 웹사이트 배포 | 간단한 웹 페이지, 포트폴리오 |
| Autoscale | 자동 확장 웹 애플리케이션 | 웹앱, API 서버, 챗봇 |
| Reserved VM | 항상 켜져 있는 전용 서버 | 항상 응답해야 하는 서비스 |

> 💡 대부분의 해커톤 프로젝트는 **Autoscale**이면 충분합니다.

* * *

## 🔧 프로젝트 시작: App vs Design

새 프로젝트를 만들면 **App**과 **Design** 두 개의 탭이 보입니다.

| 상황 | 추천 모드 |
| --- | --- |
| "고객 피드백 수집 앱 만들어줘" | **App** |
| "회사 소개 페이지 만들어줘" | **Design** |
| "데이터 대시보드 만들어줘" | **App** |

> 💡 **헷갈리면 App 모드로 시작하세요.** 해커톤에서는 기능이 동작하는 게 핵심입니다.

* * *

## 🔌 빌트인 서비스 (Settings > Integrations)

Agent에게 "데이터베이스 추가해줘"라고 말하면 끝입니다.

| 기능 | 한 줄 설명 |
| --- | --- |
| **Replit Database** | 앱 데이터를 저장하는 PostgreSQL DB (10GB 무료) |
| **Replit App Storage** | 이미지, 파일 등을 저장하는 파일 저장소 |
| **Replit Auth** | 로그인/회원가입 기능 한 번에 추가 |
| **Replit Domains** | 커스텀 도메인 연결 |

* * *

## 💡 Replit 활용 팁

- **Secrets**: API 키 같은 민감 정보를 안전하게 관리
- **실시간 협업**: Google Docs처럼 여러 명이 동시에 같은 프로젝트 작업 가능
- **GitHub 연동**: 버전 관리 및 코드 백업
