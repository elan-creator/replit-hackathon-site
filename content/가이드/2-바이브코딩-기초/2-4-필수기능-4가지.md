# 2.4 바이브코딩 필수 역량 4가지

> 코딩 실력이 아니라 이 4가지 "메타 스킬"이 바이브코딩의 핵심입니다

* * *

## 🎯 한 줄 정의

**바이브코딩을 잘하려면 코딩을 배울 필요 없이, 이 4가지 역량만 갖추면 됩니다.**

* * *

## 📋 4가지 필수 역량

### 1️⃣ 프롬프트 설계 — "무엇을, 누구를 위해, 어떤 순서로"

AI에게 명확하게 요구사항을 전달하는 능력.

```
❌ "좋은 앱 만들어줘"
✅ "팀 PM이 매주 사용할 회의록 액션아이템 추출 앱. 
    입력: 텍스트 → 출력: 담당자별 액션아이템 목록"
```

**핵심**: 프롬프트는 **상사에게 업무 보고**하듯 쓰면 됩니다. 목적, 대상, 범위, 완료 기준.

> 💡 오늘 배운 5가지 요소(목적, 사용자, 입력, 출력, 제약)를 기억하세요. 이 구조만 지키면 AI가 엉뚱한 결과를 만들 확률이 크게 줄어듭니다.

### 2️⃣ 오류 수정 루프 — "에러는 실패가 아니라 정보다"

에러를 두려워하지 말고, 빠르게 고치는 리듬을 만드는 능력.

```
에러 발견 → 에러 메시지 복사 → Agent에게 전달 → 수정 확인 → 반복
```

**핵심**: **에러 메시지 = AI에게 줄 수 있는 가장 정확한 디버그 정보**. 에러가 나면 오히려 기회입니다 — 구체적인 수정 요청이 가능하니까요.

### 3️⃣ 버전/롤백 관리 — "돌아갈 길을 확보하고 실험"

변경 전에 현재 상태를 기억해두고, 문제 시 되돌리는 능력.

Replit에서 롤백하는 3가지 방법:

| 방법 | 언제 쓰나 | 어떻게 |
| --- | --- | --- |
| Agent에게 "취소해줘" | 직전 변경을 되돌릴 때 | "방금 변경 취소해줘" |
| Replit History | 특정 시점으로 돌아갈 때 | 좌측 메뉴 → History |
| Git 커밋 | 중요 마일스톤 저장 | Agent에게 "커밋해줘" |

**핵심**: "마음껏 실험하되, 돌아갈 길은 확보"

### 4️⃣ 배포 후 검증 — "만드는 것보다 검증이 더 중요"

만든 앱을 실제 사용자 관점에서 테스트하는 능력.

```
나만 테스트 (Level 1) → 시나리오 테스트 (Level 2) → 타인 테스트 (Level 3)
```

**핵심**: **"다른 사람이 설명 없이 쓸 수 있는가?"** — 이 질문에 "네"라고 답할 수 있으면 성공입니다.

* * *

## 📊 4가지 역량 자가 진단

| 역량 | 오늘 이전 | 오늘 목표 |
| --- | --- | --- |
| 프롬프트 설계 | 모호한 요청 | **5가지 요소 포함 + 예시** |
| 오류 수정 | 에러에 당황 | **에러 메시지 복사 → Agent 전달** |
| 버전 관리 | 변경 후 후회 | **큰 변경 전 백업 의식** |
| 배포 검증 | "되는 것 같은데" | **시나리오 테스트 + 타인 피드백** |

> 💡 오늘 해커톤에서 이 4가지를 **한 번씩이라도 경험**했으면 성공입니다. 완벽할 필요 없어요 — 루프를 돌아본 경험 자체가 가장 중요합니다.
