# 2.2 실습: UX 보완 & 기능 추가

> Part 1의 피드백을 반영하고, 실사용 가능한 수준으로 앱을 업그레이드합니다

* * *

## 🎯 한 줄 정의

**MVP 고도화 = 동작하는 앱을 "실제로 쓸 만한" 앱으로 업그레이드하는 과정**

* * *

## 왜 필요한가?

Part 1에서 만든 MVP는 "핵심 기능이 돌아가는" 상태입니다. 하지만 실제로 쓰려면 **안정성과 사용성**이 뒷받침되어야 합니다. 이 시간에 받은 피드백을 반영하고, 엣지 케이스를 처리하고, UX를 개선합니다.

| Part 1 MVP 🔨 | Part 2 고도화 ✨ |
| --- | --- |
| 핵심 기능 동작 확인 | 다양한 입력에서도 안정 동작 |
| 기본 디자인 | 사용하기 편한 UX |
| 에러 시 빈 화면 | 에러 시 안내 메시지 표시 |
| "나만" 쓸 수 있는 수준 | "다른 사람도" 바로 쓸 수 있는 수준 |

* * *

## 📋 따라하기 (35분)

### Phase 1: 피드백 기반 개선 (15분)

Part 1에서 받은 피드백 3개를 **우선순위 순서대로** 반영합니다.

#### 🔥 피드백 반영 프롬프트 템플릿 (복붙용)

```
현재 [문제 상황]입니다.
[원하는 상태]로 수정해줘.

예시:
- "현재 결과가 텍스트로만 나옵니다. 
   담당자별로 카드 형태로 구분해서 보여줘"
- "현재 빈 입력을 넣으면 에러가 나요. 
   빈 입력 시 '내용을 입력해주세요' 메시지를 보여줘"  
- "현재 결과를 복사할 방법이 없어요.
   결과 아래에 '복사하기' 버튼을 추가해줘"
```

**핵심 규칙**: 한 번에 수정 요청 **1개만!** 여러 개를 동시에 요청하면 AI가 예상치 못한 방식으로 변경할 수 있습니다.

```
❌ "디자인 바꾸고, 기능 추가하고, 에러 처리도 해줘"
✅ "먼저 빈 입력 에러 처리 추가해줘" 
   → 확인 후: "이제 결과를 카드 형태로 바꿔줘"
   → 확인 후: "복사 버튼 추가해줘"
```

### Phase 2: 핵심 기능 안정화 (10분)

Must 기능 3개가 **다양한 입력에서도 정상 동작**하는지 확인합니다.

```
테스트 시나리오:
1. 정상 입력 → 정상 출력 확인 ✅
2. 긴 입력 (텍스트 500자 이상) → 성능 문제 없는지 확인
3. 빈 입력 → "입력해주세요" 메시지가 나오는지 확인
4. 특수문자 포함 입력 → 깨지지 않는지 확인
5. 엔터/줄바꿈 포함 → 제대로 처리되는지 확인
```

문제 발견 시:
```
Agent에게: "[어떤 입력]을 넣으면 [현재 결과]가 나옵니다.
[원하는 결과]로 처리하도록 수정해줘"
```

### Phase 3: UX 개선 (10분)

시간이 남으면 사용성을 개선합니다. **아래 목록에서 1~2개만 골라서** 적용하세요.

#### 자주 쓰는 UX 개선 프롬프트 (복붙용)

| 개선 항목 | 프롬프트 |
| --- | --- |
| 로딩 표시 | "처리 중일 때 로딩 스피너와 '처리 중...' 텍스트를 보여줘" |
| 결과 복사 | "결과 영역 아래에 '클립보드에 복사' 버튼을 추가해줘" |
| 레이아웃 | "전체 레이아웃을 중앙 정렬로, 최대 너비 800px로 제한해줘" |
| 색상/폰트 | "메인 색상을 파란색(#3B82F6)으로, 폰트 크기를 16px로 바꿔줘" |
| 빈 상태 | "데이터가 없을 때 '아직 결과가 없습니다' 안내 문구를 보여줘" |
| 입력 안내 | "입력창에 placeholder로 입력 예시를 보여줘" |
| 반응형 | "모바일 화면(375px)에서도 잘 보이도록 반응형으로 만들어줘" |

## 💡 시간 관리 주의

```
❌ 디자인에 20분 쓰기 → 핵심 기능 불안정한 채로 남음
✅ 핵심 기능 안정화에 20분 → 남은 시간에 UX 개선

❌ 새 기능 계속 추가 → 기존 기능 깨짐
✅ 기존 기능 완성도 높이기 → 시간 남으면 1개만 추가
```

**우선순위 정리:**
1. 🔴 **에러 수정** — 동작 안 하는 것부터
2. 🟡 **안정성** — 다양한 입력에서도 동작
3. 🟢 **사용성** — 쓰기 편하게, 보기 좋게
4. ⚪ **추가 기능** — 시간 남으면만

## ⚠️ 트러블슈팅 FAQ

### "수정하다가 원래 되던 기능이 깨졌어요"

```
Agent에게: "방금 변경 취소해줘. [원래 되던 기능]이 깨졌어"
```
또는 Replit의 **History** 기능에서 이전 버전으로 돌아가세요.

### "Agent가 수정을 자꾸 엉뚱한 방식으로 해요"

```
더 구체적으로 요청하세요:
❌ "결과를 예쁘게 해줘"
✅ "결과를 카드 형태로 보여줘. 각 카드에는 담당자 이름(굵게), 
    할 일, 기한이 표시되어야 해. 카드 사이에는 간격 16px"
```

### "기능을 추가하고 싶은데 시간이 부족해요"

**"Nice to have" 목록에 적어두고**, 2주 실행 계획에 포함하세요. 오늘은 핵심 기능의 안정성이 우선입니다.
