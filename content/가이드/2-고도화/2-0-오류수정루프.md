# 2.0 오류 수정 루프

> AI 앱 개발에서 가장 중요한 스킬: 에러를 두려워하지 않고 빠르게 고치는 것

* * *

## 🎯 한 줄 정의

**오류 수정 루프 = 에러 발견 → 에러 메시지 복사 → Agent에게 전달 → 수정 확인 → 반복**

* * *

## 왜 필요한가?

Part 1에서 만든 MVP는 "동작은 하지만" 다양한 상황에서 에러가 날 수 있습니다. **에러는 실패가 아닙니다.** 바이브코딩에서 에러는 **AI에게 보낼 수 있는 가장 정확한 디버그 정보**입니다.

| 에러에 당황하면 😰 | 에러를 활용하면 ✨ |
| --- | --- |
| "안 돼요..." → 멈춤 | "이 에러 고쳐줘" → 해결 |
| 뭐가 문제인지 모름 | 에러 메시지가 원인을 알려줌 |
| 처음부터 다시 만들려고 함 | 기존 코드에서 해당 부분만 수정 |

* * *

## 📋 오류 수정 루프 (25분 실습)

### 기본 루프: 4단계

```
1. 에러 발견 → "버튼 클릭하면 아무 반응이 없어"
     ↓
2. 에러 메시지 확인 → 화면에 빨간 글씨 또는 콘솔 확인
     ↓
3. Agent에게 전달 → "이 에러가 나요: [에러 메시지 복붙]. 수정해줘"
     ↓
4. 수정 확인 → 같은 동작 다시 테스트
     ↓
(에러 남아있으면 2번으로 돌아가기)
```

> 💡 **이 루프가 빨라지면 = 바이브코딩 실력이 올라간 것입니다.** 에러를 얼마나 두려워하느냐가 아니라, 얼마나 빨리 고치느냐가 실력입니다.

### Step 1: 에러 메시지 찾는 방법

에러 메시지는 **3곳**에서 찾을 수 있습니다:

| 위치 | 확인 방법 | 보이는 것 |
| --- | --- | --- |
| **화면** | 앱에 빨간 글씨가 뜨면 | 사용자가 볼 수 있는 에러 |
| **Replit 콘솔** | 화면 하단 Console 탭 | 서버 에러, 실행 에러 |
| **브라우저 콘솔** | `F12` → Console 탭 | JavaScript 에러 |

> 💡 **가장 쉬운 방법**: 에러가 나면 **화면에 보이는 에러 메시지를 스크린샷**하거나 텍스트 복사해서 Agent에게 보내세요. 콘솔까지 확인하면 금상첨화!

### Step 2: Agent에게 에러 보고하는 방법

#### 🔥 에러 보고 프롬프트 템플릿 (복붙용)

**상황 1: 화면에 에러가 보일 때**
```
[어떤 동작을 했을 때] 이런 에러가 나요:
[에러 메시지 그대로 복사]
수정해줘.
```

**상황 2: 아무 반응이 없을 때**
```
[어떤 버튼]을 클릭하면 아무 반응이 없어요.
클릭하면 [원하는 동작]이 되어야 하는데, 
화면에 변화가 없습니다. 수정해줘.
```

**상황 3: 결과가 이상할 때**
```
[어떤 동작]을 하면 [현재 결과]가 나오는데,
[원하는 결과]로 바꿔줘.

예: "추출 버튼을 누르면 영어로 결과가 나오는데, 한국어로 바꿔줘"
```

**상황 4: 여러 에러가 동시에 있을 때**
```
⚠️ 한 번에 1개씩만 요청하세요!

❌ "에러 5개 다 고쳐줘" → AI 혼란
✅ "먼저 이 에러 고쳐줘: [에러 1]" → 해결 후 다음 에러
```

### Step 3: 수정 확인

Agent가 수정한 후:

1. **같은 동작**을 다시 테스트 → 에러 사라졌는지 확인
2. **다른 입력**으로도 테스트 → 같은 에러가 안 나는지 확인
3. **다른 기능**도 테스트 → 수정하다가 다른 곳이 깨지지 않았는지 확인

* * *

## 🧪 실습: 자주 나오는 에러 유형별 대처법

### 유형 1: "빈 화면 / 앱이 안 뜸"

```
Agent에게: "앱 실행하면 빈 화면만 나와요. 콘솔에 이 에러가 있습니다:
[콘솔 에러 복사]. 수정해줘"
```

### 유형 2: "버튼이 안 먹힘"

```
Agent에게: "[버튼 이름] 버튼을 클릭해도 반응이 없습니다.
이 버튼을 클릭하면 [원하는 동작]이 되어야 합니다. 수정해줘"
```

### 유형 3: "데이터가 안 저장됨"

```
Agent에게: "폼에 데이터를 입력하고 제출하면 저장이 안 됩니다.
페이지를 새로고침하면 입력한 데이터가 사라집니다. 
데이터가 유지되도록 수정해줘"
```

### 유형 4: "한국어가 깨짐 / 영어로 나옴"

```
Agent에게: "앱의 모든 텍스트를 한국어로 바꿔줘.
버튼, 라벨, 안내 메시지, 에러 메시지 전부 한국어로."
```

### 유형 5: "레이아웃이 깨짐 / 디자인이 이상함"

```
Agent에게: "[어디]의 레이아웃이 깨져 있어요.
[원하는 형태]로 수정해줘.
예: 카드가 한 줄에 3개씩 나오도록"
```

## 💡 프로 팁: 수정 요청 잘하는 법

### 1. "뭐가 문제인지" + "어떻게 되어야 하는지" 같이 말하기

```
❌ "이거 이상해요"
✅ "추출 결과에 날짜가 안 나와요. '김철수: 보고서 작성 (기한: 3/15)' 
    형식으로 날짜도 표시해줘"
```

### 2. 큰 변경은 작은 단계로 나눠서

```
❌ "디자인 전체 리뉴얼해줘"
✅ "먼저 헤더 색상을 파란색(#3B82F6)으로 바꿔줘"
   → 확인 후: "본문 폰트 크기를 16px로 키워줘"
   → 확인 후: "버튼을 둥글게 만들어줘"
```

### 3. 변경 전 상태를 기억해두기

```
💡 큰 변경 전에 Agent에게:
"지금 상태를 커밋해줘" 또는 현재 동작 상태를 스크린샷

→ 수정 후 문제 발생 시: "방금 변경 취소해줘" (롤백)
```
