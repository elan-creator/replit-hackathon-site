# Replit 소개

> Replit = 브라우저에서 코드 작성, 실행, 배포까지 한 번에 되는 올인원 개발 플랫폼

* * *

## 🎯 한 줄 정의

**Replit은 복잡한 설치 없이 브라우저만 있으면 앱을 만들고 배포할 수 있는 클라우드 기반 개발 환경입니다.**

* * *

## 왜 Replit인가? — 개발 장벽을 허물다

### 전통적 개발 과정 (비개발자에게 너무 어려움)

```
코드 에디터 설치 (VS Code 등) → 프로그래밍 언어 설치 (Python, Node.js 등) 
→ 패키지 매니저 설치 (pip, npm) → 프레임워크 설치 (React, Django 등) 
→ 환경변수 설정 → 로컬 서버 실행 → 코드 작성 → 빌드 → 서버 설정 → 도메인 연결 → 배포
```

**문제점**: 이 모든 과정에 **초보자 기준 반나절~하루, 숙련자도 수십 분**이 걸립니다. 특히 비개발자에게는 진입 장벽이 너무 높습니다. 대부분의 아이디어가 "환경 설정" 단계에서 좌절됩니다.

### Replit 개발 과정 (누구나 5분 안에 시작)

```
replit.com 접속 → 로그인 → Create Repl 클릭 → "이런 앱 만들어줘" (Agent 모드) → Deploy 클릭
```

**소요 시간**: 5분 이내. Replit은 이 복잡한 모든 과정을 웹 브라우저 안으로 가져왔습니다.

## 📋 Replit의 핵심 기능

Replit은 개발자와 비개발자 모두에게 혁신적인 기능을 제공합니다.

### 0. 프로젝트 시작 화면 — App vs Design

Replit에서 새 프로젝트를 만들면 가장 먼저 보이는 화면입니다. 상단에 **App**과 **Design** 두 개의 탭이 있고, 하단에 **Build**, **Web app**, **Auto** 등의 옵션이 보입니다.

#### App 모드 🔷
- **용도**: 실제 동작하는 웹 앱, API 서버, 자동화 도구 등을 만들 때
- **특징**: 백엔드 로직, 데이터베이스 연동, API 호출, 사용자 인증 등 **기능 중심**
- **예시**: 할 일 관리 앱, 대시보드, 챗봇, 블로그 시스템, 업무 자동화 도구
- **해커톤에서**: 대부분의 과제는 App 모드로 진행합니다

#### Design 모드 🎨
- **용도**: 시각적인 디자인을 먼저 잡고 싶을 때, UI/UX 프로토타입을 만들 때
- **특징**: 레이아웃, 색상, 폰트, 버튼 배치 등 **디자인 중심**으로 시작
- **예시**: 랜딩페이지, 포트폴리오, 제품 소개 페이지, 이벤트 페이지
- **활용 팁**: 디자인 모드에서 시작해도 나중에 기능(로직)을 추가할 수 있어요

#### 언제 뭘 선택할까?

| 상황 | 추천 모드 |
| --- | --- |
| "고객 피드백 수집 앱 만들어줘" | **App** |
| "우리 회사 소개 페이지 만들어줘" | **Design** |
| "데이터 대시보드 만들어줘" | **App** |
| "이벤트 초대 페이지 예쁘게 만들어줘" | **Design** |
| "할 일 관리 + 알림까지 되는 도구" | **App** |

> 💡 **헷갈리면 App 모드로 시작하세요.** 해커톤에서는 기능이 동작하는 게 핵심이니까요. 디자인은 나중에 "이 화면 더 예쁘게 바꿔줘"라고 Agent에게 요청하면 됩니다.

#### 하단 옵션 설명

- **Build 드롭다운**: 프로젝트 빌드 방식 선택 (대부분 기본값 사용)
- **Web app / 기타**: 프로젝트 유형 선택 — Web app, Mobile app, Game 등에서 선택. 해커톤에서는 **Web app**이 기본
- **Auto**: AI 모델 설정 — Auto로 두면 Replit이 상황에 맞는 최적의 모델을 자동 선택합니다

#### 프로젝트 유형 선택

새 프로젝트 생성 시 하단에서 프로젝트 유형을 선택할 수 있습니다:

| 유형 | 설명 | 예시 |
| --- | --- | --- |
| 🌐 **Web app** | 브라우저에서 돌아가는 웹 애플리케이션 | 대시보드, 블로그, 관리 도구, 쇼핑몰 |
| 📱 **Mobile app** | 모바일 앱 (React Native 등) | iOS/Android 앱 |
| ▶️ **Animation** 🆕 | 애니메이션/모션 그래픽 | 인터랙티브 애니메이션, 시각 효과 |
| 📊 **Data app** | 데이터 분석/시각화 앱 | 차트 대시보드, 리포트 자동화, 데이터 탐색 |
| 🎮 **3D Game** | 3D 게임 | 브라우저 기반 3D 게임 |
| 🔄 **Automation** | 반복 업무 자동화 도구 | 이메일 자동 발송, 스프레드시트 동기화, 슬랙 알림 |
| 📄 **Start from scratch** | 빈 프로젝트에서 시작 | 직접 언어/프레임워크를 선택하고 싶을 때 |

> 💡 **해커톤에서는 대부분 Web app을 선택하면 됩니다.** 업무 자동화 과제라면 Automation, 데이터 분석이라면 Data app도 좋은 선택입니다. 헷갈리면 Web app으로 시작하세요!

### 1. Replit Agent (AI 기반 앱 생성)

AI가 자연어 설명만으로 전체 앱을 만들어줍니다. 여러분이 해커톤에서 사용한 핵심 기능입니다.

-   **자연어로 요청**: "할 일 관리 웹앱 만들어줘. 할 일 추가, 완료 체크, 삭제가 가능해야 해"
-   **전체 앱 코드 생성**: 파일 구조, 의존성, 기본 코드까지 Agent가 알아서 처리
-   **상호작용적 개발**: 중간에 Agent가 질문하면 답변하며 함께 앱을 만들어감
-   **자동 디버깅**: 에러 발생 시 자동으로 디버깅 시도 및 수정 제안

> 💡 **비개발자는 "무엇을" 만들지에 집중하고, "어떻게" 만드는지는 Agent에게 맡기면 됩니다.**

### 2. 즉시 실행 (Instant Run)

코드를 작성하거나 Agent가 앱을 생성하면, **별도의 빌드 과정 없이 즉시 결과물을 확인할 수 있습니다.**

-   **실시간 미리보기**: 웹앱의 경우, Replit 화면 내에서 실시간으로 앱이 동작하는 모습을 보여줌
-   **빠른 피드백 루프**: 수정 즉시 결과를 확인하며 빠르게 개선 가능
-   **에러 즉시 감지**: 문제가 발생하면 즉시 화면이나 콘솔에 에러 메시지 표시

### 3. 원클릭 배포 (Deploy)

만든 앱을 인터넷에 공개하여 누구나 URL로 접속할 수 있게 합니다. 단 한 번의 클릭으로!

| 배포 유형 | 설명 | 적합한 경우 |
| --- | --- | --- |
| Static | 정적 웹사이트 배포 (HTML, CSS, JavaScript만) | 간단한 웹 페이지, 포트폴리오, 블로그 |
| Autoscale | 서버가 포함된 자동 확장 웹 애플리케이션 | 웹앱, API 서버, 챗봇 |
| Reserved VM | 항상 켜져 있어야 하는 전용 서버 | 항상 응답해야 하는 중요한 서비스, 게임 서버 |

> 💡 대부분의 해커톤 프로젝트는 **Autoscale**이면 충분합니다. Replit은 배포된 앱에 자동으로 URL을 부여하고 SSL 인증서까지 발급해줍니다.

### 4. 실시간 협업 (Multiplayer)

Google Docs처럼 여러 명이 동시에 같은 프로젝트에서 작업할 수 있습니다.

-   **코드 공동 편집**: 팀원들이 동시에 코드 작성 및 수정
-   **공유 개발 환경**: 모든 팀원이 동일한 개발 환경을 사용하여 "내 컴퓨터에서는 되는데?" 문제 해결
-   **채팅 기능**: 프로젝트 내에서 실시간으로 대화하며 협업

## 🔧 Settings > Integrations — Replit의 빌트인 서비스들

Replit 프로젝트의 **Settings > Integrations** 화면에 가면, 별도 설정 없이 바로 사용할 수 있는 클라우드 서비스들이 있습니다. 이 서비스들은 Replit이 직접 관리해주기 때문에, 외부 서비스 가입이나 복잡한 설정 없이 Agent에게 "데이터베이스 추가해줘"라고 말하면 끝입니다.

### Replit Managed (빌트인 통합 기능)

| 기능 | 타입 | 한 줄 설명 |
| --- | --- | --- |
| **Replit Database** | PostgreSQL (SQL) | 앱의 데이터를 저장하고 관리하는 데이터베이스 |
| **Replit App Storage** | Object Storage | 이미지, 동영상, 파일 등을 저장하는 파일 저장소 |
| **Replit Auth** | Authentication | 사용자 로그인/회원가입 기능을 한 번에 추가 |
| **Replit Domains** | Domains | 앱에 커스텀 도메인(URL)을 연결 |

#### 📦 Replit Database (PostgreSQL)

- **뭐하는 건가요?** 앱에서 사용하는 데이터(사용자 정보, 게시글, 설정값 등)를 저장하고 불러오는 SQL 데이터베이스입니다
- **어떤 상황에서?** 앱에 "데이터 저장"이 필요한 거의 모든 경우. 회원 정보, 게시판 글, 주문 내역, 설문 응답 등
- **해커톤 활용 예시**: "고객 피드백을 받아서 저장하는 앱" → Agent에게 "데이터베이스에 피드백을 저장해줘"라고 하면 자동으로 테이블 생성 + 코드 연동까지 해줌
- **특징**: 10GB 무료 저장 공간 제공, Agent 체크포인트로 데이터베이스 상태를 되돌릴 수 있음 (타임 트래블!), 워크스페이스에서 직접 쿼리 실행 및 데이터 확인 가능

#### 🗂️ Replit App Storage (Object Storage)

- **뭐하는 건가요?** 이미지, 동영상, 문서 같은 **파일**을 클라우드에 저장하는 서비스입니다. 데이터베이스가 "표 형태의 데이터"를 저장한다면, App Storage는 "파일 그 자체"를 저장합니다
- **어떤 상황에서?** 사용자가 사진을 업로드하거나, 파일을 다운로드하거나, 미디어를 관리해야 할 때
- **해커톤 활용 예시**: "팀 사진 공유 앱" → 사용자가 올린 사진을 App Storage에 저장하고 갤러리로 보여주기
- **특징**: 개발 환경과 배포 환경에서 같은 파일을 공유, Python/JavaScript SDK 제공, Agent에게 "파일 업로드 기능 추가해줘"라고 하면 자동 설정

#### 🔐 Replit Auth (Authentication)

- **뭐하는 건가요?** 앱에 로그인/회원가입 기능을 추가해줍니다. 이메일 로그인, Google/Apple/GitHub 소셜 로그인까지 지원합니다
- **어떤 상황에서?** "누가 이 앱을 쓰는지" 알아야 하거나, 사용자별로 다른 화면을 보여줘야 할 때
- **해커톤 활용 예시**: "팀별 할 일 관리 앱" → 로그인한 사용자만 자기 팀의 할 일을 볼 수 있게
- **특징**: Firebase + Google Cloud Identity Platform 기반의 엔터프라이즈급 보안, reCAPTCHA 봇 차단 내장, Agent에게 "로그인 기능 추가해줘"라고 한 마디면 완료

#### 🌐 Replit Domains (Custom Domains)

- **뭐하는 건가요?** 배포된 앱의 URL을 `내앱이름.replit.app` 대신 `www.my-app.com` 같은 나만의 도메인으로 바꿀 수 있습니다
- **어떤 상황에서?** 앱을 정식으로 공개하거나, 브랜드 이름으로 URL을 만들고 싶을 때
- **해커톤 활용 예시**: 발표용 데모를 `우리팀이름.com`으로 연결하면 더 전문적으로 보입니다
- **특징**: SSL 인증서(https) 자동 발급, 기본 제공되는 `*.replit.app` 서브도메인은 무료, 커스텀 도메인은 도메인 구매 비용만 별도

### Connectors (외부 서비스 연동)

Replit은 외부 서비스와도 쉽게 연동할 수 있습니다. **Connectors**를 통해 Google Sheets, Slack, Stripe 등 다양한 서비스를 앱에 붙일 수 있어요.

- **Google Sheets**: 스프레드시트 데이터를 앱에서 읽고 쓰기
- **Slack**: 앱에서 슬랙 메시지 보내기/받기
- **Stripe**: 결제/구독 기능 연동
- **BigQuery, Databricks, Snowflake**: 데이터 웨어하우스 연결

> 💡 Agent에게 "Google Sheets에서 데이터 가져와서 대시보드 만들어줘"라고 하면 Connector 설정부터 코드 작성까지 알아서 해줍니다.

### Import to Replit (프로젝트 가져오기)

이미 다른 곳에서 작업 중인 프로젝트가 있다면, Replit으로 가져올 수 있습니다:

| 옵션 | 설명 |
| --- | --- |
| **GitHub** | GitHub 저장소를 통째로 가져오기 (Rapid Import 또는 Guided Import) |
| **Bitbucket** | Bitbucket 저장소 가져오기 |
| **Figma Design** | 피그마 디자인을 실제 동작하는 React 앱으로 변환 |
| **Lovable** | Lovable에서 만든 프로젝트를 GitHub 경유로 가져오기 |
| **Bolt** | Bolt에서 만든 프로젝트를 GitHub 경유로 가져오기 |
| **Vercel** | Vercel 프로젝트의 GitHub 저장소를 선택해서 가져오기 |
| **Zip file** | .zip 파일을 직접 업로드해서 프로젝트 시작 |

> 💡 **해커톤 팁**: Figma에서 디자인한 UI가 있다면 Figma Import로 바로 코드로 변환할 수 있어요! 디자이너와 협업할 때 매우 유용합니다.

## 🧩 Developer Frameworks (프로젝트 템플릿)

`Start from scratch`를 선택하면 다양한 프레임워크와 언어 템플릿을 고를 수 있습니다. 각 템플릿에는 필요한 환경이 미리 세팅되어 있어서, 설치 걱정 없이 바로 코딩을 시작할 수 있습니다.

### Websites (웹 프레임워크)

웹 앱을 만들 때 사용하는 프레임워크 템플릿입니다:

## 💡 Replit 활용 팁

-   **Version Control (Git)**: Replit은 GitHub와 연동하여 버전 관리를 할 수 있습니다. `git commit`, `git push` 명령어를 터미널에서 직접 사용하거나, Agent에게 "현재 변경사항을 커밋해줘"라고 요청할 수 있습니다.
-   **Secrets**: API 키나 비밀번호 같은 민감한 정보를 안전하게 관리할 수 있는 기능입니다. 환경 변수로 쉽게 접근할 수 있습니다.
-   **Database**: 작은 프로젝트를 위한 Key-Value 데이터베이스(`Replit DB`)를 기본 제공합니다. 복잡한 DB 설정 없이 간단한 데이터 저장이 가능합니다.

## 🚀 Replit의 미래

Replit은 단순한 개발 도구를 넘어, "AI를 통한 소프트웨어 개발"의 미래를 제시하고 있습니다. 누구나 아이디어만 있으면 앱을 만들고 배포할 수 있는 세상을 꿈꿉니다.

-   **Ghostwriter (코드 자동 완성)**: 현재 코드를 분석하여 다음 코드를 추천해주는 AI 기능
-   **Code Repair (코드 수정)**: 에러가 발생했을 때 AI가 자동으로 코드를 분석하고 수정 제안
-   **Test Case Generation (테스트 코드 생성)**: 작성한 코드에 대한 테스트 케이스를 AI가 자동으로 생성

> 💡 이 모든 기능은 여러분이 더 빠르게, 더 쉽게 앱을 만들 수 있도록 돕습니다. Replit은 코딩을 "전문가의 영역"에서 "모두의 영역"으로 가져오고 있습니다.
