# GitHub 연동

> Replit 프로젝트를 GitHub에 연결하여 코드를 안전하게 관리합니다

* * *

## 🎯 한 줄 정의

**GitHub 연동 = Replit에서 만든 코드를 GitHub에 자동으로 백업하고 버전 관리하는 것**

* * *

## 왜 필요한가?

| GitHub 없이 😰 | GitHub 연동 ✨ |
| --- | --- |
| Replit에서만 코드 존재 | 코드가 GitHub에도 저장 |
| 실수로 삭제하면 복구 어려움 | 언제든 이전 버전 복구 가능 |
| 혼자만 코드 접근 가능 | 팀원과 코드 공유 용이 |
| 포트폴리오로 활용 어려움 | GitHub 프로필에 프로젝트 표시 |

> 💡 **해커톤 당일에는 필수가 아닙니다.** 해커톤 후 프로젝트를 이어갈 때 설정하면 됩니다.

* * *

## 📋 따라하기

### Step 1: GitHub 계정 준비

1. [github.com](https://github.com) 계정이 없다면 생성
2. 이미 있다면 로그인 상태 확인

### Step 2: Replit에서 GitHub 연결

1. Replit 프로젝트 열기
2. 좌측 사이드바에서 **Git** 아이콘 (분기 모양) 클릭
3. **Connect to GitHub** 클릭
4. GitHub 계정 인증 → 권한 허용

### Step 3: Repository 생성

1. **Create a new GitHub repository** 선택
2. Repository 이름 입력 (예: `meeting-action-items`)
3. Public 또는 Private 선택
4. **Create** 클릭

### Step 4: 코드 Push

1. 변경사항이 있으면 좌측 Git 패널에 표시
2. 커밋 메시지 입력 (예: "첫 MVP 완성")
3. **Commit & Push** 클릭

* * *

## 💡 활용 팁

### 커밋 메시지 잘 쓰기

```
✅ 좋은 예:
- "회의록 입력 기능 추가"
- "액션아이템 추출 로직 개선"
- "모바일 레이아웃 수정"

❌ 나쁜 예:
- "수정"
- "업데이트"
- "asdf"
```

### 언제 Push할까?

- 기능 1개 완성할 때마다
- 큰 변경 전에 (안전망 확보)
- 하루 작업 마무리할 때
