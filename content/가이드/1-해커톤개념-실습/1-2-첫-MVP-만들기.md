# 1.2 첫 MVP 만들기

> Replit Agent에게 프롬프트를 주고, 40분 안에 동작하는 앱을 만듭니다

* * *

## 🎯 한 줄 정의

**첫 MVP = 문제정의 캔버스의 Must 기능 3개가 동작하는 최소 앱**

* * *

## 왜 "최소"인가?

| 완벽한 앱을 목표로 😰 | 최소 앱을 목표로 ✨ |
| --- | --- |
| 40분 안에 완성 불가 | 40분 안에 동작하는 앱 확보 |
| 중간에 방향 전환 어려움 | 빠르게 피봇 가능 |
| 프롬프트가 길고 복잡 | 프롬프트가 짧고 명확 |

> 💡 **MVP(Minimum Viable Product)** = 최소 기능 제품. "이것만 되면 가치 있다"는 수준의 앱입니다. 디자인이 별로여도 상관없어요 — **동작하는 것이 핵심**입니다.

* * *

## 📋 따라하기 (50분 = 프롬프트 작성 10분 + 빌드 40분)

### Step 1: Replit에서 새 프로젝트 생성 (2분)

1. [replit.com](https://replit.com) 로그인
2. **Create Repl** 클릭
3. **Agent** 선택 (직접 코딩 X, AI가 만들어줌)

> 💡 반드시 **Agent 모드**를 선택하세요. 일반 Repl은 직접 코드를 써야 하지만, Agent 모드는 자연어로 설명하면 AI가 전체 앱을 만들어줍니다.

### Step 2: 프롬프트 작성 (8분)

문제정의 캔버스를 바탕으로 프롬프트를 작성합니다. **아래 템플릿을 그대로 복사해서 빈칸만 채우세요.**

#### 🔥 프롬프트 템플릿 (복붙용)

```
다음 웹앱을 만들어줘:

[앱 이름]: [한 줄 설명]

대상 사용자: [누가 쓰는지]

핵심 기능:
1. [Must 기능 #1]
2. [Must 기능 #2]
3. [Must 기능 #3]

제약사항:
- 한국어 UI
- 심플하고 깔끔한 디자인 (tailwind CSS 사용)
- 로그인 기능 불필요
- 모바일에서도 볼 수 있으면 좋음

입력 예시:
[실제로 넣을 데이터 예시를 적어주세요]

출력 예시:
[원하는 결과 형태를 적어주세요]
```

#### ✅ 좋은 프롬프트 예시

```
다음 웹앱을 만들어줘:

회의록 액션아이템 추출기: 회의록 텍스트를 넣으면 액션아이템을 
자동으로 추출해주는 도구

대상 사용자: 팀 PM

핵심 기능:
1. 회의록 텍스트를 붙여넣을 수 있는 큰 입력창
2. "추출" 버튼을 누르면 액션아이템을 자동 분류
3. 담당자별로 그룹핑해서 보여주고, 복사 버튼으로 클립보드 복사

제약사항:
- 한국어 UI
- 심플하고 깔끔한 디자인
- 로그인 기능 불필요
- 결제 기능 불필요

입력 예시:
"3/10 마케팅 회의. 참석: 김철수, 이영희.
 김철수가 보고서를 3/15까지 완성하기로 함.
 이영희는 디자인 시안을 3/12까지 공유 예정."

출력 예시:
- 김철수: 보고서 완성 (기한: 3/15)
- 이영희: 디자인 시안 공유 (기한: 3/12)
```

#### ❌ 나쁜 프롬프트 예시

```
❌ "좋은 앱 만들어줘"
   → AI가 뭘 만들지 모름

❌ "로그인, 결제, 대시보드, 알림, 검색, 통계, 
    관리자 페이지, 사용자 관리 다 넣어줘"
   → 범위가 무한대 → 아무것도 제대로 안 됨

❌ "잘 만들어줘"
   → 성공 기준이 없으니 AI도 감으로 만듦
```

#### 💡 좋은 프롬프트의 5가지 요소

| 요소 | 설명 | 예시 |
| --- | --- | --- |
| **목적** | 이 앱이 뭘 하는지 | "회의록에서 액션아이템을 추출" |
| **사용자** | 누가 쓰는지 | "팀 PM" |
| **입력** | 무엇을 넣는지 | "회의록 텍스트 붙여넣기" |
| **출력** | 무엇이 나오는지 | "담당자별 액션아이템 목록" |
| **제약** | 하지 말아야 할 것 | "로그인 기능 불필요" |

> 💡 **프롬프트는 상사에게 업무 보고하듯** 쓰면 됩니다. 목적, 대상, 범위, 완료 기준 — 업무 요청을 잘 하는 사람이 프롬프트도 잘 씁니다.

### Step 3: Agent가 만드는 것 지켜보기 (15분)

Replit Agent가 앱을 만드는 동안:

1. **진행 상황을 지켜보세요** — Agent가 질문하면 답변
2. **완벽하지 않아도 OK** — 일단 동작하는 게 먼저
3. **추가 요청은 나중에** — 첫 버전이 나올 때까지 기다리기

> ⚠️ **주의**: Agent에게 한꺼번에 너무 많은 것을 요청하지 마세요. 첫 프롬프트에서 핵심 기능 3개만 요청하고, 나머지는 앱이 동작한 후에 추가 요청합니다.

**Agent가 질문할 때 대응법:**

| Agent 질문 | 추천 답변 |
| --- | --- |
| "어떤 프레임워크를 사용할까요?" | "자유롭게 선택해줘" |
| "데이터베이스를 추가할까요?" | "일단 없이 만들어줘" |
| "디자인 스타일은?" | "심플하고 깔끔하게" |
| "추가 기능은?" | "일단 핵심 3개만 먼저" |

### Step 4: 첫 테스트 (5분)

앱이 생성되면 바로 테스트합니다:

```
체크리스트:
□ 앱이 에러 없이 열리는가?
□ Must 기능 #1이 동작하는가?
□ Must 기능 #2가 동작하는가?
□ Must 기능 #3이 동작하는가?
□ 한국어가 제대로 표시되는가?
```

### Step 5: 빠른 수정 (10분)

테스트에서 발견된 문제를 수정합니다. **한 번에 하나씩** 요청하세요.

#### 수정 요청 프롬프트 템플릿 (복붙용)

```
[현재 문제]를 [원하는 상태]로 수정해줘.

예시:
- "버튼을 클릭하면 에러가 나요. 콘솔에 이 에러가 뜹니다: [에러 메시지]. 수정해줘"
- "결과가 영어로 나와요. 한국어로 바꿔줘"
- "입력창이 너무 작아요. 높이를 300px 이상으로 키워줘"
- "추출 결과에 날짜가 빠져있어요. 날짜도 함께 표시해줘"
```

> 💡 **핵심 팁**: 에러가 나면 당황하지 말고, **에러 메시지를 그대로 복사**해서 Agent에게 보여주세요. "이 에러가 나요: [에러 메시지]. 수정해줘" — 이것만으로 대부분 해결됩니다.

## ⚠️ 막혔을 때 — 트러블슈팅 FAQ

### "Agent가 이상한 걸 만들어요"

**원인**: 프롬프트가 모호해서 AI가 추측으로 만든 것.

**해결**: 프롬프트를 더 구체적으로 수정해서 **새 대화**에서 다시 요청하세요. 특히 **입력/출력 예시**를 포함하면 결과가 확 달라집니다.

### "에러가 계속 나요"

**해결**: Agent에게 에러 메시지를 그대로 복사해서 보여주세요:
```
"Run 버튼을 클릭하면 이 에러가 나요:
[에러 메시지 전체 복사]
수정해줘"
```

### "Agent가 응답이 없어요 / 멈췄어요"

**해결**: 브라우저 새로고침 후 다시 시도. 그래도 안 되면 새 Repl을 만들어서 프롬프트를 다시 넣으세요.

### "40분이 다 됐는데 완성이 안 돼요"

**해결**: Must 기능 #1만이라도 동작하면 성공입니다. 나머지는 Part 2에서 이어갑니다. **완벽보다 동작이 먼저.**

### "토큰이 부족하다는 메시지가 나와요"

**해결**: 퍼실리테이터에게 알려주세요. 실습 토큰은 운영 측에서 지원합니다.
