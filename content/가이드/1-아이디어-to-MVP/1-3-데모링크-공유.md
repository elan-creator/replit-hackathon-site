# 1.3 데모 링크 공유

> 만든 앱을 링크 하나로 공유하고 피드백을 받습니다

* * *

## 🎯 한 줄 정의

**데모 링크 공유 = 내가 만든 MVP를 동료에게 보여주고 즉석 피드백을 받는 과정**

* * *

## 왜 필요한가?

혼자 만들고 혼자 보면 끝입니다. **공유하는 순간** 진짜 학습이 시작됩니다.

| 공유 안 하면 😰 | 공유하면 ✨ |
| --- | --- |
| "나는 잘 되는 것 같은데..." | "이 부분이 헷갈려요" → 즉시 개선 포인트 발견 |
| 혼자만 아는 프로토타입 | 팀원이 직접 써볼 수 있는 앱 |
| 해커톤 끝나면 잊혀짐 | Part 2에서 개선할 목록 확보 |

* * *

## 📋 따라하기 (10분)

### Step 1: Replit에서 배포하기 (3분)

1. Replit 프로젝트 상단의 **Deploy** 버튼 클릭
2. 배포 유형 선택 (Agent가 안내해줌 — 대부분 **Autoscale** 선택)
3. 배포 완료 후 **URL 복사**

> 💡 Replit은 클릭 한 번으로 배포가 끝납니다. 별도 서버 설정, 도메인 구매 등이 필요 없어요.

> ⚠️ **Deploy 버튼이 안 보이면?** Agent에게 "이 앱을 배포해줘"라고 요청하세요. Agent가 배포 설정을 도와줍니다.

### Step 2: 링크 공유 (2분)

배포된 링크를 공유하세요:

- 🔗 운영팀이 준비한 **공유 시트**에 링크 등록
- 💬 옆 참가자에게 직접 링크 전달
- 📱 **본인 핸드폰으로 접속**해서 모바일 테스트 (중요!)

### Step 3: 피드백 받기 (5분)

옆 참가자와 **2분씩** 서로의 앱을 체험하고 피드백합니다:

```
피드백 3줄:
1. 첫인상 (3초): 화면을 보자마자 느낀 점
2. 막힌 곳: 어디서 헷갈리거나 멈췄는지
3. "이것만 있으면": 실무에서 쓰려면 뭐가 추가되면 좋을지
```

> 💡 **피드백을 메모하세요!** Part 2에서 이 피드백을 기반으로 앱을 개선합니다.

### Step 4: Part 2 개선 목록 정리

받은 피드백 중 **Part 2에서 반영할 항목 3개**를 선택하세요:

```
개선 우선순위:
1. _____________ (반드시 반영 — 핵심 기능 관련)
2. _____________ (시간 되면 반영 — UX 개선)
3. _____________ (2주 계획에 포함 — 추가 기능)
```
