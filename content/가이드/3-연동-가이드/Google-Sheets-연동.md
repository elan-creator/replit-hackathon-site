# Google Sheets 연동

> 📌 **해커톤 이후 참고 자료** — 이 가이드는 해커톤 당일이 아닌, 이후에 활용하기 위한 자료입니다.

> Replit 앱에서 Google Sheets를 데이터 저장소로 활용합니다

* * *

## 🎯 한 줄 정의

**Google Sheets 연동 = 데이터베이스 지식 없이, 익숙한 Google Sheets를 앱의 데이터 저장소로 활용하는 것**

* * *

## 왜 필요한가?

복잡한 데이터베이스(DB)를 직접 구축하고 관리하는 것은 비개발자에게 큰 장벽입니다. Google Sheets는 누구나 쉽게 접근하고 데이터를 관리할 수 있는 도구입니다. 이 기능을 활용하면 앱에서 생성된 데이터를 자동으로 시트에 기록하거나, 시트의 데이터를 앱에서 불러와 활용할 수 있습니다.

| 기존 방식 (DB 필요) 😰 | Google Sheets 연동 ✨ |
| --- | --- |
| DB 설치, 설정, 쿼리문 학습 | 시트 생성, API 연동만 하면 끝 |
| 전문 개발자 도움 필수 | 비개발자도 직접 구현 가능 |
| 데이터 확인 시 DB 도구 필요 | 웹 브라우저로 시트 열면 끝 |

### 📊 활용 사례

-   **신청자 관리 시스템**: 앱에서 제출된 해커톤 신청 정보를 Google Sheets에 자동으로 기록
-   **고객 피드백 수집**: 앱에서 받은 사용자 피드백을 실시간으로 시트에 축적
-   **데모 링크 모음**: 해커톤 참가자별 앱 링크를 한 곳에 정리하여 관리
-   **설문 데이터 연동**: Google Forms 응답 데이터를 앱에서 불러와 시각화하거나 분석
-   **간단한 CMS (콘텐츠 관리 시스템)**: 앱에 표시될 공지사항이나 FAQ 내용을 시트에서 관리

* * *

## 📋 따라하기: Google Sheets 연동 (단계별 가이드)

Google Sheets 연동은 "구글 클라우드 플랫폼"에서 API를 활성화하고, "서비스 계정"을 통해 권한을 부여하는 과정이 필요합니다. 다소 복잡해 보일 수 있지만, 한 번 설정해두면 모든 Replit 프로젝트에서 재활용할 수 있습니다.

### Step 1: Google Sheets 준비 (5분)

1.  [sheets.google.com](https://sheets.google.com)에서 새 시트 생성
2.  시트의 첫 행에 **헤더(컬럼명)** 입력 (예: `날짜`, `앱이름`, `제출자`, `링크`, `피드백`)
3.  시트 이름을 기억해두세요 (예: `Replit_Hackathon_Submissions`)
4.  **시트 URL**을 복사해두세요 (예: `https://docs.google.com/spreadsheets/d/<스프레드시트ID>/edit`)

> 💡 시트의 첫 행에 정확한 헤더를 입력하는 것이 중요합니다. 이 헤더가 앱에서 데이터를 읽고 쓰는 기준이 됩니다.

### Step 2: Google Cloud Platform (GCP) 프로젝트 생성 (10분)

1.  [console.cloud.google.com](https://console.cloud.google.com) 접속
2.  좌측 상단 프로젝트 선택 드롭다운 → **"새 프로젝트"** 클릭
3.  프로젝트 이름 입력 (예: `Replit-Sheets-API`) → **"만들기"** 클릭

> ⚠️ **주의**: GCP는 유료 서비스지만, Google Sheets API는 대부분의 사용량에서 무료 등급을 제공합니다. 소액이라도 과금될 수 있으니 주의하세요. 보통은 무료 등급으로 충분합니다.

### Step 3: Google Sheets API 활성화 (5분)

1.  GCP 대시보드에서 방금 만든 프로젝트 선택
2.  좌측 메뉴 "API 및 서비스" → **"라이브러리"** 클릭
3.  검색창에 `Google Sheets API` 입력 → 검색 결과 클릭
4.  **"사용 설정"** 버튼 클릭

### Step 4: 서비스 계정 생성 및 키 다운로드 (10분)

서비스 계정은 Replit 앱이 Google Sheets에 접근할 수 있도록 하는 "봇 계정"입니다.

1.  GCP 대시보드 → 좌측 메뉴 "API 및 서비스" → **"사용자 인증 정보"** 클릭
2.  상단 **"사용자 인증 정보 만들기"** → **"서비스 계정"** 선택
3.  서비스 계정 이름 입력 (예: `replit-sheet-writer`) → **"만들고 계속하기"**
4.  **역할 선택** (선택 사항이지만 안전을 위해 지정 권장) → **"기본"** → **"프로젝트"** → **"편집자"** 선택 (Sheets에 쓰고 읽기 위함)
5.  "계속" → "완료"
6.  생성된 서비스 계정 클릭 → 상단 **"키"** 탭 선택
7.  **"키 추가"** → **"새 키 만들기"** → **"JSON"** 선택 → **"만들기"**
8.  `서비스_계정_이름.json` 파일이 다운로드됩니다. **이 파일을 안전하게 보관하세요!** (절대 외부에 노출 금지)

> 💡 다운로드한 JSON 파일 안에 **`client_email`** 이 있습니다. 이 이메일 주소가 시트에 대한 권한을 가질 서비스 계정의 주소입니다.

### Step 5: Google Sheets에 서비스 계정 권한 부여 (3분)

1.  Step 1에서 만든 Google Sheets로 돌아가기
2.  우측 상단 **"공유"** 버튼 클릭
3.  "사용자 및 그룹과 공유" 입력창에 Step 4에서 확인한 **`client_email`** 주소를 붙여넣기
4.  권한을 **"편집자"**로 설정 → **"공유"** 버튼 클릭

> 💡 이 과정을 생략하면 Replit 앱에서 시트에 접근할 수 없습니다.

### Step 6: Replit Secrets에 키 등록 (5분)

다운로드한 JSON 키 파일을 Replit 프로젝트의 "Secrets"에 안전하게 저장합니다.

1.  Replit 프로젝트 접속 (Agent 모드)
2.  좌측 메뉴 자물쇠 모양 **"Secrets"** 아이콘 클릭
3.  "New Secret" 버튼 클릭
4.  `Key`에는 `GOOGLE_SERVICE_ACCOUNT_KEY` 입력
5.  `Value`에는 다운로드한 JSON 파일의 **내용 전체**를 복사해서 붙여넣기 ( `{ ... }` 포함)
6.  "Add new secret" 클릭

> ⚠️ **매우 중요**: JSON 파일 내용을 직접 복사해서 붙여넣어야 합니다. 파일 자체를 업로드하는 것이 아닙니다. 그리고 이 정보는 절대 코드에 직접 노출해서는 안 됩니다.

### Step 7: Replit 코드에서 Sheets 연동 (Agent에게 요청)

이제 Agent에게 Google Sheets를 연동해달라고 요청할 수 있습니다.

#### 🔥 Agent 요청 프롬프트 템플릿 (복붙용)

```
내 Replit 프로젝트에 Google Sheets 연동 기능을 추가해줘.

시트 이름: [Step 1에서 기억해둔 시트 이름]
스프레드시트 ID: [시트 URL에서 "d/"와 "/edit" 사이 문자열]

다음 기능을 추가해줘:
1. [예: 폼 제출 시 시트에 새 행 추가]
2. [예: 시트의 모든 데이터를 불러와서 웹앱에 표시]
```


## ⚠️ 트러블슈팅 FAQ

### "시트에 데이터가 안 써져요 / 안 읽어져요"

**해결**: 다음을 확인하세요.
1.  Step 5에서 서비스 계정(`client_email`)에 **"편집자" 권한**을 부여했는지?
2.  Step 6에서 Secrets 키 이름(`GOOGLE_SERVICE_ACCOUNT_KEY`)과 JSON 내용이 정확한지?
3.  Step 1에서 시트 이름과 헤더가 정확한지? (띄어쓰기, 대소문자)

### "GCP 프로젝트를 만들었는데 API 활성화가 안 돼요"

**해결**: GCP 대시보드에서 현재 프로젝트가 제대로 선택되었는지 확인하세요. 간혹 다른 프로젝트가 선택되어 있을 수 있습니다.

### "JSON 키 파일을 잃어버렸어요"

**해결**: Step 4부터 다시 진행하여 새로운 서비스 계정 키를 생성해야 합니다. 기존 키는 복구할 수 없습니다.

### "Agent가 자꾸 코딩 에러를 내요"

**해결**: Sheets API는 다소 복잡할 수 있습니다. Agent에게 에러 메시지를 정확히 복사해서 보여주세요. 필요한 경우, Agent에게 "Node.js Google Sheets API 샘플 코드를 이용해서 연동해줘"와 같이 더 구체적인 기술 스택을 지정해볼 수 있습니다.
