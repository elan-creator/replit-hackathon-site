# Replit 4시간 해커톤 커리큘럼

> 4시간 안에 아이디어를 실행 가능한 앱으로 만드는 해커톤

* * *

## 🎯 개요

| 항목 | 내용 |
| --- | --- |
| **시간** | 4시간 (240분) |
| **대상** | 비개발자 포함 현업팀 (PM, 디자이너, 운영, 기획, 마케팅, 리더) |
| **목표** | 참가자 전원이 **실행 가능한 MVP 링크**를 확보 |
| **도구** | Replit (브라우저만 있으면 OK) |

* * *

## 💡 이 해커톤이 특별한 이유

| 기존 해커톤 😰 | 이 해커톤 ✨ |
| --- | --- |
| 8시간 이상 소요 | **4시간 압축형** |
| 개인 Pro 계정 선결제 필요 | **실습 토큰 비용 운영 측 지원** |
| 로컬 개발환경 셋업 필요 | **브라우저 + Replit 계정으로 즉시 시작** |
| 배포까지 복잡한 과정 | **클릭 한 번으로 배포 자동화** |
| 결과물 없이 끝나는 경우 多 | **전원 MVP 링크 확보가 목표** |

> 💡 **"코딩을 전혀 몰라도 참가할 수 있나요?"** — 네! 여러분의 업무 지식이 곧 최고의 프롬프트입니다. Replit Agent가 코드를 만들고, 여러분은 **무엇을 만들지**에 집중하면 됩니다.

## 📌 세션별 핵심 포인트

### Part 0: 시작 — "오늘 뭘 하는지 알기"

- 해커톤의 목표와 마인드셋 공유
- Replit이 뭔지 3분 만에 파악 (브라우저 올인원 개발 환경)
- **핵심**: 로컬 설치 없음, 배포 자동화, AI가 코드 작성

### Part 1: 아이디어 → 첫 MVP — "일단 돌아가게"

- **문제정의 캔버스** 1장 완성 → 대상 사용자, 핵심 문제, Must 기능 3개
- **스코프 제한이 핵심** → "이것만 되면 된다"를 명확히
- 좋은 프롬프트 vs 나쁜 프롬프트 비교 학습
- Replit Agent에게 첫 프롬프트 → **동작하는 앱 1개** 확보
- 바로 링크 공유 → 옆 사람의 실시간 피드백

### Part 2: 고도화 — "실제로 쓸 수 있게"

- 에러가 났을 때 당황하지 않고 **에러 메시지 → Agent 전달 → 수정 확인** 루프 학습
- Part 1 피드백 기반으로 UX 개선 & 기능 추가
- **배포 후 즉시 테스트** → 모바일에서도 접속 확인

### Part 3: 마무리 — "가져갈 것 확인"

- 1인 1데모 (30초~1분)
- 2주 실행 계획 수립 (개선 항목 3개)

## 📊 결과물

참가자 각자가 가져가는 것:

- ✅ **MVP 링크 1개 이상** — 실제 동작하는 웹앱
- ✅ **바이브코딩 경험** — AI로 앱 만드는 전체 프로세스 체득
- ✅ **프롬프트 작성 노하우** — 복붙 가능한 템플릿
- ✅ **2주 실행 계획** — 개선할 항목 3개 + 타임라인
