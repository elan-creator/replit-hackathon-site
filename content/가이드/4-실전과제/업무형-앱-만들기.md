## 🎯 한 줄 정의

## 왜 필요한가? — 해커톤의 경험을 실제 업무로

오늘 4시간 동안 여러분은 "코딩 없이 앱 만들기"를 경험했습니다. 이 경험은 여러분의 업무 방식에 큰 변화를 가져올 수 있습니다. 이제 여러분 주변의 실제 업무 문제를 찾아, Replit Agent를 활용하여 해결하는 것에 도전해보세요.

| 해커톤 (오늘) 🎓 | 실전 과제 (미래) 🚀 |
| --- | --- |
| 정해진 시간, 주제 | 자유로운 시간, 주제 선택 |
| 학습 & 경험 중심 | 실제 업무 문제 해결 중심 |
| 최소 MVP 완성 목표 | 기능 고도화 및 팀 확산 목표 |

## 📋 과제 예시

어떤 앱을 만들어야 할지 막막하다면, 아래 예시들을 참고하여 여러분의 업무에 맞는 아이디어를 구체화해보세요.

### 예시 1: 리포트 자동 요약기

| 항목 | 내용 |
| --- | --- |
| **문제** | 매주 팀 보고서를 수작업으로 읽고 핵심 내용을 요약하는 데 2시간 이상 소요 |
| **솔루션** | 긴 보고서 텍스트를 입력하면, 핵심 요약과 주요 액션아이템을 자동으로 추출하여 정리해주는 웹앱 |
| **Must 기능** | ① 텍스트 입력창 ② 자동 요약 버튼 ③ 요약 결과 표시 |
| **고도화** | 요약 결과 복사 버튼, 요약 길이 조절, 특정 키워드 추출 |
| **난이도** | ⭐⭐ (중급) |

### 예시 2: 회의록 액션아이템 추출기

| 항목 | 내용 |
| --- | --- |
| **문제** | 회의 후 액션아이템 정리를 깜빡하거나, 방대한 회의록에서 누가 뭘 해야 할지 파악하기 어려움 |
| **솔루션** | 회의록 텍스트를 입력하면, 담당자별로 액션아이템과 기한을 자동으로 분류하여 보여주는 웹앱 |
| **Must 기능** | ① 회의록 텍스트 입력창 ② 액션아이템 추출 버튼 ③ 담당자별 분류 결과 표시 |
| **고도화** | Google Sheets 자동 저장, Slack/Lark 담당자 알림, 기한 임박 하이라이트 |
| **난이도** | ⭐⭐ (중급) |

### 예시 3: 팀 일정 리스크 대시보드

| 항목 | 내용 |
| --- | --- |
| **문제** | 여러 프로젝트 일정을 수동으로 취합하여, 마감일 임박 리스크를 늦게 감지함 |
| **솔루션** | 팀원별 일정 데이터를 입력하면, 지연 리스크가 있는 일정을 시각적으로 표시해주는 대시보드 |
| **Must 기능** | ① 일정 데이터 입력/편집 ② 리스크 자동 계산 ③ 대시보드 시각화 |
| **고도화** | Google Calendar 연동, 지연 시 Slack/Lark 알림, 팀원별 진척도 표시 |
| **난이도** | ⭐⭐⭐ (고급) |

### 예시 4: 고객 FAQ 응답 보조 도구

| 항목 | 내용 |
| --- | --- |
| **문제** | 반복되는 고객 질문에 CS 담당자가 매번 수동으로 답변을 찾아 보내야 함 |
| **솔루션** | 기존 FAQ 데이터를 기반으로 고객 질문에 가장 적합한 답변 초안을 자동으로 생성해주는 웹앱 |
| **Must 기능** | ① FAQ 데이터 입력/관리 ② 고객 질문 입력 ③ 적합한 답변 초안 생성 |
| **고도화** | 답변 만족도 피드백, 유사 질문 추천, 고객 문의 채널 직접 연동 |
| **난이도** | ⭐⭐⭐ (고급) |

## 📊 평가 기준 (루브릭)

만든 앱의 가치를 객관적으로 평가할 수 있는 기준입니다.

| 평가 항목 | 1점 (미흡) | 2점 (보통) | 3점 (우수) |
| --- | --- | --- | --- |
| **문제 적합성** | 문제가 모호하거나 실제 업무와 동떨어짐 | 문제가 명확하지만 범위가 넓거나 중요도가 낮음 | 구체적이고 실제 업무에 시급하며, 가치 있는 문제 해결 |
| **사용성** | 앱 사용법을 별도로 설명해야 함 | 대부분 직관적이지만 일부 기능이 헷갈림 | 설명 없이도 누구나 바로 이해하고 사용할 수 있음 |
| **실행 가능성** | 에러가 빈번하고 불안정 | 핵심 흐름은 동작하지만 엣지 케이스에서 에러 | 안정적으로 동작하며 에러 처리 포함 |
| **확장 가능성** | 현재 기능으로 끝 | 추가 기능 아이디어가 있지만 계획 미흡 | 구체적인 확장 계획과 연동 가능성까지 고려 |

## 💡 실전 과제 성공 팁

1. **작은 문제부터 시작**: "이거 하나만 해결되면 정말 편하겠다" 싶은 작은 문제부터 시작하세요.
2. **"나 자신"이 첫 사용자**: 본인이 직접 가장 많이 쓰는 앱이 가장 좋은 앱이 될 가능성이 높습니다.
3. **반복해서 개선**: 한 번 만들고 끝이 아니라, 꾸준히 피드백을 반영하고 고도화하세요.
4. **팀에 공유하고 피드백 받기**: 혼자 쓰기보다 팀원들과 공유하며 실제 업무에 적용해보세요.
5. **Replit의 기능 활용**: Google Sheets 연동, Slack/Lark 알림 연동 등을 적극 활용해보세요.
