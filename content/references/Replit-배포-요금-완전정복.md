# Replit 배포 요금 완전정복 🚀

> "내 서비스 월 얼마 나올까?" — 이 문서를 읽고 나면 스스로 계산할 수 있습니다.

해커톤에서 만든 앱을 세상에 공개하려면 **배포(Deployment)**가 필요합니다. Replit은 배포까지 원클릭으로 해결할 수 있는 플랫폼이지만, "공짜인 줄 알았는데 요금이 나왔어요!"라는 당황스러운 상황을 피하려면 요금 구조를 정확히 이해해야 합니다.

이 문서는 **2026년 3월 기준** Replit 공식 문서의 요금 정보를 바탕으로 작성되었습니다.

## 1. 배포 유형 한눈에 보기

Replit은 4가지 배포 유형을 제공합니다. 각각의 특성이 완전히 다르니, 내 앱에 맞는 걸 고르는 게 첫 번째 절약 포인트입니다.

### 📊 배포 유형별 비교표

| 구분 | Static | Autoscale | Reserved VM (Shared) | Reserved VM (Dedicated) |
|------|--------|-----------|---------------------|------------------------|
| **한 줄 설명** | HTML/CSS/JS만 서빙 | 트래픽에 따라 자동 확장/축소 | 항상 켜진 공유 서버 | 항상 켜진 전용 서버 |
| **백엔드 지원** | ❌ 없음 | ✅ 있음 | ✅ 있음 | ✅ 있음 |
| **월 기본 요금** | 무료 | $1/월 | $20/월 | $40~$160/월 |
| **트래픽 없을 때** | 데이터 전송 없으면 $0 | $0 (잠자기 모드) | 그래도 $20 나감 | 그래도 전액 나감 |
| **Cold Start** | 없음 (CDN) | 있음 (수 초) | 없음 (항상 ON) | 없음 (항상 ON) |
| **적합한 경우** | 포트폴리오, 랜딩페이지 | 일반 웹앱, API | 챗봇, 항상 켜져야 하는 서비스 | 고성능 프로덕션 |

### Reserved VM 사양별 가격

| 유형 | vCPU | RAM | 월 가격 |
|------|------|-----|---------|
| Shared Medium | 0.5 vCPU | 2GB | **$20/월** |
| Dedicated Small | 1 vCPU | 4GB | **$40/월** |
| Dedicated Medium | 2 vCPU | 8GB | **$80/월** |
| Dedicated Large | 4 vCPU | 16GB | **$160/월** |

> 💡 **핵심 포인트**: Reserved VM은 서버가 24시간 돌아가기 때문에 트래픽이 없어도 요금이 나갑니다. 반면 Autoscale은 아무도 안 쓰면 $1/월만 나갑니다.

## 2. Autoscale 심층 분석 — 가장 많이 쓰게 될 유형

해커톤에서 만드는 대부분의 앱은 **Autoscale 배포**를 사용하게 됩니다. 가장 유연하고, 소규모일 때 가장 저렴하기 때문입니다.

### 💰 Autoscale 요금 구성 (3가지 요소)

| 요소 | 단가 | 설명 |
|------|------|------|
| **Base Fee** | $1/월 | 배포를 유지하는 기본 비용 |
| **Compute Units** | $3.20 / 백만 CU | 서버가 일한 만큼 |
| **Requests** | $1.20 / 백만 건 | 요청 수만큼 |

### Compute Unit(CU)이란?

Compute Unit은 서버가 "얼마나 열심히 일했는지"를 측정하는 단위입니다.

계산 공식:
- **CPU 1초 사용 = 18 CU**
- **메모리 1GB를 1초 사용 = 2 CU**

예를 들어, 어떤 요청을 처리하는 데 CPU 0.5초 + 메모리 0.5GB를 2초 사용했다면:
- CPU: 0.5초 × 18 = 9 CU
- 메모리: 0.5GB × 2초 × 2 = 2 CU
- **합계: 11 CU**

### Cold Start란?

Autoscale 앱은 아무도 안 쓰면 **잠자기 모드**로 들어갑니다 (15분 무활동 후). 이 상태에서 누군가 접속하면:

```
1. 사용자가 URL 접속 → 2. 서버 깨어남 (Cold Start, 2~5초) → 3. 페이지 로딩
```

- **장점**: 안 쓸 때 요금 $0
- **단점**: 첫 번째 방문자가 2~5초 기다려야 함 (이후 방문자는 즉시)

> 15분 안에 다음 요청이 오면 서버가 깨어 있으므로 Cold Start 없음!

### Max Instances 설정

트래픽이 몰리면 Replit이 자동으로 서버를 여러 개 띄웁니다. 하지만 서버가 많아지면 요금도 올라갑니다.

- **Max Instances**: 최대 몇 개까지 서버를 띄울지 제한
- 해커톤 앱이라면 **1~2개**면 충분
- 이걸 설정 안 하면 트래픽 폭주 시 요금 폭탄 가능!

### 실제 과금 타임라인

```
시간   00:00   00:01   00:02   ...   00:15   00:16
상태   [잠자기]  [깨어남]  [요청처리]  ...  [대기중]  [다시 잠자기]
과금    $0     과금시작   과금중    ...   과금중    $0
```

- 요청이 들어온 순간부터 마지막 요청 처리 완료까지만 과금
- 15분 동안 요청이 없으면 자동으로 잠자기 → 과금 중지
- 여러 요청이 동시에 들어오면 같은 서버에서 처리 (중복 과금 아님)

## 4. 경제성 판단 기준 — 언제 다른 플랫폼을 고려해야 할까?

### ✅ Replit이 유리한 경우

- **프로토타입 & MVP**: 코딩부터 배포까지 원스톱. 인프라 설정 시간 = 0
- **소규모 서비스 (MAU 5,000 이하)**: 월 $1~$5로 운영 가능
- **트래픽 변동이 큰 서비스**: Autoscale이 자동으로 대응
- **비개발자/1인 개발자**: 서버 관리 지식 불필요
- **해커톤/교육**: 빠른 배포, 쉬운 공유

### ❌ 다른 플랫폼이 유리한 경우

- **MAU 5만 이상**: Compute Unit 비용이 빠르게 증가
- **정적 사이트 대량 트래픽**: Vercel/Netlify가 무료 또는 더 저렴
- **복잡한 인프라**: DB, 큐, 캐시 등 → AWS/GCP가 유연
- **글로벌 엣지 배포**: Vercel/Cloudflare Workers가 더 적합
- **비용 최적화가 중요한 단계**: Railway/Render가 GB당 더 저렴

### 📊 플랫폼 비교표 (소규모 웹앱 기준)

| 플랫폼 | 무료 티어 | 소규모 (DAU 100) | 중규모 (DAU 1,000) | 특징 |
|--------|----------|-----------------|-------------------|------|
| **Replit** | Starter (1앱, 30일) | ~$2~3/월 | ~$20~27/월 | 코딩+배포 올인원 |
| **Vercel** | 무료 (취미) | 무료~$20/월 | $20/월 (Pro) | 프론트엔드 최적화 |
| **Railway** | $5 무료 크레딧 | ~$5/월 | ~$10~20/월 | 풀스택 심플 |
| **Render** | 무료 (750시간) | 무료~$7/월 | ~$7~25/월 | Docker 지원 |
| **AWS (EC2 t3.micro)** | 12개월 무료 | ~$8~10/월 | ~$8~10/월 | 무한 확장 가능 |

### 🎯 MAU별 플랫폼 추천

| MAU | 추천 플랫폼 | 이유 |
|-----|-----------|------|
| 0~1,000 | **Replit** | 개발 속도가 핵심, 비용 거의 없음 |
| 1,000~5,000 | **Replit 또는 Railway** | 둘 다 합리적, Replit은 편의성 우위 |
| 5,000~50,000 | **Railway / Render** | 가격 대비 성능이 좋아짐 |
| 50,000+ | **AWS / GCP + Vercel** | 본격적인 인프라 최적화 필요 |

## 5. 비용 폭탄 방지 팁 🛡️

해커톤에서 만든 앱이 갑자기 인기를 얻으면? 기쁘지만 요금이 무서울 수 있습니다. 미리 방지합시다!

### ① Budget(예산) 설정하기

Replit은 **월간 지출 한도**를 설정할 수 있습니다.

1. **Settings → Account → Billing** 접속
2. **Spending alerts** 설정 — 특정 금액 도달 시 이메일 알림
3. 크레딧 팩 구매 시 한도를 정해서 구매

> ⚠️ 크레딧이 다 떨어지면 앱이 중지될 수 있으니, 약간의 여유를 두세요.

### ② Max Instances 제한 (필수!)

Autoscale 배포 설정에서:

- **해커톤/데모**: Max Instances = **1**
- **소규모 서비스**: Max Instances = **2**
- **중규모 이상**: 트래픽에 맞게 조절하되, 무제한은 절대 금지!

### ③ Static으로 전환할 수 있는지 확인

다음 조건에 해당하면 **Static 배포로 전환**해서 비용을 거의 0으로 줄일 수 있습니다:

- 서버에서 데이터를 처리하지 않는 경우 (순수 프론트엔드)
- 외부 API만 호출하는 SPA (React, Vue 등)
- 포트폴리오, 문서 사이트, 랜딩페이지

> ⚠️ **주의**: Replit Agent로 만든 앱은 풀스택이라 Static 배포가 안 됩니다. 수동으로 프론트/백 분리가 필요합니다.

### ④ 모니터링 방법

- **Usage Dashboard**: [replit.com/usage](https://replit.com/usage)에서 실시간 사용량 확인
- **배포 로그**: 배포된 앱의 Publishing 탭에서 로그와 상태 확인
- **습관화**: 매일 한 번 Usage 페이지 체크 (특히 배포 첫 주)

### ⑤ 크레딧 팩 할인 활용

추가 크레딧이 필요하면 대량 구매 시 할인:

| 크레딧 팩 | 가격 | 할인 |
|-----------|------|------|
| $100 | $100 | — |
| $300 | $290 | $10 할인 |
| $500 | $480 | $20 할인 |
| $1,000 | $950 | $50 할인 |

## 7. 내 서비스 비용 직접 계산하기 🧮

아래 공식으로 내 앱의 예상 월 비용을 계산해보세요:

### Autoscale 월 비용 공식

```
월 비용 = $1 (기본) 
        + (총 요청 수 ÷ 1,000,000) × $1.20
        + (총 Compute Units ÷ 1,000,000) × $3.20
```

### Compute Unit 빠른 추정

| 앱 유형 | 요청당 평균 CU | 참고 |
|---------|---------------|------|
| 간단한 페이지 렌더링 | 3~5 CU | HTML 반환만 |
| API 응답 (DB 조회) | 8~15 CU | 간단한 CRUD |
| AI API 호출 포함 | 20~50 CU | 외부 API 대기 시간 |
| 이미지 처리 | 50~200 CU | CPU 집약적 |

### 계산 예시

"하루 200명이 각 10번 요청하는 SaaS, 요청당 평균 10 CU"

```
월 요청 수 = 200 × 10 × 30 = 60,000건
월 CU = 60,000 × 10 = 600,000 CU

월 비용 = $1 + (60,000 / 1,000,000 × $1.20) + (600,000 / 1,000,000 × $3.20)
       = $1 + $0.07 + $1.92
       = $2.99/월
```

**스타벅스 아메리카노 한 잔보다 싸다!** ☕

## 8. 핵심 요약

1. **백엔드 없으면 Static** → 거의 무료
2. **대부분의 해커톤 앱은 Autoscale** → 월 $1~$5
