# 레플릿 에이전트 질문지

> 아래 질문을 레플릿 에이전트 채팅에 복붙하면, 프로젝트의 구현 방식을 상세하게 확인할 수 있습니다.
> 답변을 수업자료에 반영하면 실습생들에게 훨씬 풍부한 레퍼런스를 제공할 수 있습니다.

## 📌 replit.mfitlab.com 질문

### 기술 스택 & 아키텍처
1. 이 프로젝트의 전체 기술 스택을 알려줘. (프론트엔드, 백엔드, DB, 배포)
2. 프로젝트 폴더 구조를 보여줘. 주요 파일과 역할을 설명해줘.
3. 어떤 프레임워크를 사용했어? (React, Next.js, Flask 등)
4. 라우팅 구조가 어떻게 되어 있어?

### 블로그 시스템
5. 블로그 탭의 글 목록은 어떻게 관리돼? (DB? 파일? CMS?)
6. 직접 쓴 글과 본사 번역 글은 어떻게 구분하고 있어?
7. 글 작성/수정/삭제 플로우가 어떻게 되어 있어?
8. 마크다운으로 쓰는 건지, 에디터가 있는 건지?

### 데이터베이스 & 저장소
9. 어떤 DB를 사용하고 있어? 테이블/컬렉션 구조를 보여줘.
10. Replit DB를 쓰는 건지, 외부 DB(Supabase, Firebase 등)를 쓰는 건지?
11. 이미지나 파일은 어디에 저장돼?

### 외부 API 연동
12. 외부 API를 사용하는 게 있어? (결제, 인증, 이메일 등)
13. API 키는 어떻게 관리하고 있어? (환경변수?)
14. CORS나 보안 설정은 어떻게 했어?

### 배포 & 운영
15. 커스텀 도메인(mfitlab.com)은 어떻게 연결했어?
16. 배포 프로세스가 어떻게 되어 있어? (자동 배포?)
17. 트래픽이 몰릴 때 스케일링은 어떻게 돼?
18. 월 운영 비용이 얼마나 들어? (Replit 요금제 기준)

### 디자인 & UX
19. CSS 프레임워크 뭐 써? (Tailwind, Bootstrap 등)
20. 반응형(모바일) 대응은 어떻게 했어?
21. 다크모드 지원해? 구현 방식은?
