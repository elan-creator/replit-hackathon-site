| 구분 | 기술 | 쉬운 설명 |

### 프로젝트 폴더 구조

```
replit.mfitlab.com/
├── client/                    ← 🖥️ 사용자가 보는 화면 (프론트엔드)
│   ├── src/
│   │   ├── pages/             ← 📄 각 페이지 파일들 (랜딩, 신청, 블로그, 기업용)
│   │   ├── components/        ← 🧩 재사용 가능한 화면 조각들
│   │   └── hooks/             ← 공통 기능 (모바일 감지 등)
│   └── public/                ← 🖼️ 이미지, 로고 등 정적 파일
│
├── server/                    ← ⚙️ 서버 (백엔드)
│   ├── routes.ts              ← API 경로 정의 (신청 처리, 블로그 데이터)
│   ├── email.ts               ← 이메일 발송 기능
│   └── storage.ts             ← 데이터 저장/조회 로직
│
├── shared/                    ← 🤝 프론트/백엔드 공용 데이터 구조
└── cache/                     ← 📦 블로그 데이터 캐시
```

> **쉽게 비유하면**: `client/`는 "가게 인테리어와 진열", `server/`는 "창고와 사무실", `shared/`는 "공용 매뉴얼"입니다.

## 📝 블로그 시스템

### 콘텐츠를 어떻게 가져오나?

블로그 글은 **외부 콘텐츠 서버(Writer API)**에서 가져옵니다. DB에 직접 저장하는 방식이 아닙니다.

```
[Writer API 서버] ──(데이터 요청)──> [mfitlab.com 서버] ──(화면 표시)──> [사용자 브라우저]
     ↑                                    ↓
  글 원본 저장                        캐시(임시 저장)
                                     (6시간마다 갱신)
```

**동작 방식**:
1. 서버 시작 시 Writer API에서 모든 글을 가져와 `cache/`에 임시 저장
2. 사용자 접속 시 캐시된 데이터를 빠르게 제공
3. 6시간마다 자동 갱신
4. Writer API 장애 시에도 캐시 덕분에 정상 표시

### 직접 쓴 글 vs 본사 번역 글

| 구분 | source 값 | URL 패턴 |
|------|-----------|----------|
| **본사 번역 글** | `replit-blog` | `/blog/글ID` |
| **자체 작성 글** | `marketfitlab` | `/blog/articles/키워드/슬러그` |

블로그 목록에서 두 종류가 날짜순으로 섞여 표시되며, 탭/필터로 구분할 수 있습니다.

### 글 관리 플로우

현재 글 관리는 **Writer API 서버**에서 이루어집니다:
- Writer에서 글 작성/수정/삭제 → 6시간 내 mfitlab.com에 자동 반영
- 관리자 에디터 UI는 구현되어 있지만 백엔드 연동은 프로토타입 단계

## 🔌 외부 API 연동

| 서비스 | 용도 | 비용 |
|--------|------|------|
| **Resend** | 해커톤 신청/기업 문의 이메일 알림 | 무료 (월 3,000건) |
| **Writer API** | 블로그 콘텐츠 제공 | 별도 Replit 프로젝트 |

### API 키 관리

**Replit Secrets**에 환경변수로 저장합니다:
- 코드에서 `process.env.RESEND_API_KEY`로 참조
- 실제 키 값은 암호화되어 보관
- 다른 사람이 코드를 봐도 키 값을 알 수 없음

### CORS 설정

이 프로젝트는 **별도 CORS 설정이 필요 없습니다**:
- 프론트엔드(React)와 백엔드(Express)가 같은 서버에서 동작
- 브라우저 입장에서 "같은 출처" → CORS 문제 없음
- 풀스택 구조의 장점!

## 🚀 배포 & 운영

### 커스텀 도메인 연결

1. Replit **Deploy** 탭 → **Domains** 메뉴
2. 도메인 관리 업체에서 DNS 레코드 설정
3. Replit이 자동으로 SSL 인증서 발급 (https://)
4. 추가 비용 없음 (도메인 구매 비용만 별도)

### 배포 프로세스

```
코드 수정 → "Publish" 버튼 클릭 → 자동 빌드 → 수 분 내 전 세계 배포 완료
```

CI/CD 파이프라인, GitHub Actions, Docker 같은 것 필요 없음. Replit이 모든 것을 처리합니다.

### 오토스케일링

| 상황 | 동작 | 비용 |
|------|------|------|
| 방문자 없을 때 | 서버 자동 수면 모드 | **$0** |
| 방문자 들어오면 | 수 초 내 자동 기동 | 사용한 만큼만 |
| 동시 방문자 폭증 | 최대 3 인스턴스 확장 | 사용량 비례 |
| 트래픽 줄면 | 자동 축소 | 비용도 자동 절감 |

현재 설정: **2 vCPU / 4 GiB RAM / 최대 3 인스턴스**

## 🔍 Writer와의 연동 구조

이 프로젝트는 **Writer**(블로그 자동화 시스템)와 함께 하나의 시스템으로 작동합니다:

```
[Writer (writer-elan18.replit.app)]          [mfitlab.com]
  글 작성/번역/검수/발행          →→→       블로그에 자동 반영
  백오피스 (운영자용)                        프론트엔드 (방문자용)
```

| 구분 | replit.mfitlab.com | Writer |
| --- | --- | --- |
| **용도** | 방문자용 웹사이트 | 운영자용 관리 도구 |
| **대상** | 해커톤 참가자, 기업 담당자 | 콘텐츠 운영자 |
| **핵심 가치** | 마케팅 + 정보 전달 | 자동화 + 효율화 |
| **기술 난이도** | 중급 | 중상급 (스케줄러 + API + AI) |

> Writer 시스템에 대한 자세한 내용은 [Writer 프로젝트 분석](./실제-사례-writer) 페이지를 참고하세요.

## ✅ 핵심 시사점

1. **비개발자도 2주면 풀스택 서비스를 만들 수 있다** — Replit Agent가 코드를 생성하고, 사람은 방향을 지시
2. **전체 비용 약 12만원** — 전통 외주 대비 95~98% 절감
3. **월 운영비 3~4만원** — 오토스케일로 트래픽 없으면 비용도 없음
4. **디자인/레이아웃에 가장 많은 비용** — 코드 로직보다 UI 미세 조정이 더 많은 반복 작업을 필요로 함
5. **하나의 플랫폼에서 모든 것 해결** — 서버, DB, 배포, SSL, 모니터링까지 Replit 하나로
