# Writer — B2B 블로그 콘텐츠 자동화 시스템

> Replit Agent로 만든 AI 콘텐츠 파이프라인 — 글 생성부터 번역, 검수, 발행까지 전 과정 자동화

* * *

## 🎯 한 줄 정의

**키워드 하나 입력하면 AI가 리서치 → 글 작성 → SEO 채점 → 이미지 삽입 → 검수 대기까지 자동으로 처리하는 콘텐츠 시스템**입니다.

| # | 기능 | 설명 |
|---|------|------|

## 🔄 글 작성 → 발행 전체 플로우

```
[1단계] 주제 선정
    ↓  AI가 트렌드 분석 → 키워드 2개 추천 (또는 직접 입력)

[2단계] 리서치
    ↓  Tavily로 최신 자료 검색 → 팩트(Claim) 추출

[3단계] 글 작성
    ↓  Claude가 2~3개 관점으로 글 작성 (각 1,500~2,500자)

[4단계] SEO 채점
    ↓  자동 채점 → 80점 미만이면 피드백 후 재작성 (최대 2회)

[5단계] 이미지 삽입
    ↓  Unsplash/Pexels에서 맥락 맞는 이미지 자동 삽입

[6단계] 검수함 대기
    ↓  생성된 글이 "검수함"에 들어감 (자동 발행 아님!)

[7단계] 검수 & 승인
    ↓  사람이 검토 후 "승인" 또는 "거절" 클릭

[8단계] 발행
    ↓  승인된 글이 "완성된 글" 목록과 API에 노출

[9단계] 배포
    └  Replit Publish → 프로덕션 반영 → 랜딩페이지에 표시
```

> 💡 **핵심**: 글이 자동으로 발행되지 않습니다. 반드시 사람이 검수함에서 승인해야 합니다. AI가 생성하고, 사람이 최종 확인하는 구조입니다.

## 🤖 AI/번역 자동화

### 본사 블로그 번역 파이프라인

**3단계로 동작합니다:**

**1단계 — 크롤링**
- blog.replit.com에서 글 목록 가져오기
- 이미 번역된 글과 비교해 새 글만 식별
- 제목, 본문, 이미지, 영상, 태그 추출

**2단계 — 번역**
- 이미지/영상/코드블록 위치를 `[IMAGE-0]`, `[VIDEO-0]` 마커로 보존
- 긴 글(10,000자 이상)은 `##` 기준으로 6,000자 단위 분할 번역
- Claude API로 한국어 번역 (스타일 가이드 적용)
- 번역 후 마커를 실제 미디어로 복원

**3단계 — 저장**
- 번역된 글을 디렉토리에 저장, 원본 이미지 로컬 다운로드
- 메타데이터(원본 URL, 날짜, 태그) 기록

> **자동 스케줄**: 매주 일/수 22:00 KST. 새 글 있으면 번역, 없으면 건너뜀.

### Claude API 프롬프트 구조

**글 생성 시:**
```
[System 프롬프트]
├── 정체성: "B2B 테크 블로그 전문 작가"
├── 팩트 예산: 리서치에서 추출한 출처 있는 주장 목록
│   → "이 목록에 없는 사실은 절대 만들어내지 마세요"
├── SEO 규칙: 프론트매터, 제목, 소제목 구조
├── 금지어: "혁신적인", "획기적인" 등 과장 표현 금지
└── 이미지 규칙: 본문 중 이미지 위치 지정

[User 프롬프트]
└── "다음 키워드로 B2B 블로그 글을 작성하세요: {키워드}"
```

**번역 시:**
```
[System 프롬프트]
├── 역할: "B2B 테크 블로그 번역 전문가"
├── 번역 스타일 가이드
│   ├── 어조: 합니다체 (격식체)
│   ├── 기술 용어: 영문 유지 (Replit, Agent, CI/CD)
│   └── 첫 등장 용어: 괄호 병기 — "바이브 코딩(Vibe Coding)"
└── 앵커 규칙: [IMAGE-N], [VIDEO-N] 위치 이동 금지
```

## 📊 작업 분포 (git 히스토리 기반)

34개 실질 커밋의 카테고리별 분류:

```
■■■■■■■■■■■■■■■■■■■■■  이미지 처리/검증    21%  ← 가장 많은 반복 작업!
■■■■■■■■■■■■■■■■■■     번역 기능           18%
■■■■■■■■■■■■■■■        글 생성 파이프라인    15%
■■■■■■■■■■■■           API/연동            12%
■■■■■■■■■■■■           검수함/관리          12%
■■■■■■■■■■■■           스케줄링/배포        12%
■■■■■■                 UI/레이아웃           6%
■■■■■■                 문서                  6%
```

### 비용이 많이 든 작업 Top 3

**1위: 이미지 자동 삽입 (21%)** — 이미지 품질은 "보면 알 수 있지만 규칙으로 정의하기 어려운" 영역. AI에게 검증 기준을 점진적으로 구체화하는 반복 작업이 많았음.

**2위: 번역 파이프라인 (18%)** — "동작은 하지만 품질이 안 나오는" 문제가 반복. 스타일 가이드를 만들고 프롬프트를 튜닝하는 데 시간 소요.

**3위: 글 생성 + 브랜드 전략 (15%)** — 시스템 프롬프트가 매우 길고 복잡. 원하는 톤과 품질이 나올 때까지 프롬프트 반복 수정.

| **긴 글이 잘린다** | 섹션 단위로 분할하여 각각 처리 후 합치기 |
| **이미지가 맥락과 안 맞다** | 검증 로직을 추가하고 기준을 점진적으로 구체화 |
| **배포했는데 새 글이 안 보인다** | 개발/프로덕션 환경이 분리되어 있음 → Publish 필요 |

## ✅ 핵심 시사점

1. **초기 개발 후, 글 1편당 260원으로 자동 생산** — 초기 비용은 있지만 운영 비용은 매우 저렴
2. **사람의 검수 단계**가 있어 AI 품질 문제를 안전하게 관리
3. **이미지/번역 품질 개선에 가장 많은 반복 작업**이 필요 — "동작"보다 "품질"이 진짜 과제
4. **하나의 키워드에서 2~3편씩 다관점 생성** → 콘텐츠 볼륨 확보
5. **mfitlab.com과 JSON API로 연동** → 프론트엔드와 백엔드를 분리해 안전하게 운영
