## 2. 배포 유형별 비용

Replit은 4가지 배포 유형을 제공합니다. 내 앱에 맞는 걸 고르는 게 첫 번째 절약 포인트입니다.

| 구분 | Static | Autoscale | Reserved VM (Shared) | Reserved VM (Dedicated) |
|------|--------|-----------|---------------------|------------------------|
| **한 줄 설명** | HTML/CSS/JS만 서빙 | 트래픽에 따라 자동 확장/축소 | 항상 켜진 공유 서버 | 항상 켜진 전용 서버 |
| **백엔드 지원** | ❌ 없음 | ✅ 있음 | ✅ 있음 | ✅ 있음 |
| **월 기본 요금** | 무료 | $1/월 | $20/월 | $40~$160/월 |
| **트래픽 없을 때** | $0 | $0 (잠자기 모드) | 그래도 $20 나감 | 그래도 전액 나감 |
| **Cold Start** | 없음 (CDN) | 있음 (수 초) | 없음 (항상 ON) | 없음 (항상 ON) |
| **적합한 경우** | 포트폴리오, 랜딩페이지 | 일반 웹앱, API | 챗봇, 항상 켜져야 하는 서비스 | 고성능 프로덕션 |

### Reserved VM 사양별 가격

| 유형 | vCPU | RAM | 월 가격 |
|------|------|-----|---------|
| Shared Medium | 0.5 vCPU | 2GB | **$20/월** |
| Dedicated Small | 1 vCPU | 4GB | **$40/월** |
| Dedicated Medium | 2 vCPU | 8GB | **$80/월** |
| Dedicated Large | 4 vCPU | 16GB | **$160/월** |

> 💡 **핵심 포인트**: Reserved VM은 서버가 24시간 돌아가기 때문에 트래픽이 없어도 요금이 나갑니다. 반면 Autoscale은 아무도 안 쓰면 $1/월만 나갑니다.

## 3. Autoscale 심층 분석

해커톤에서 만드는 대부분의 앱은 **Autoscale 배포**를 사용하게 됩니다.

### 💰 요금 구성 (3가지 요소)

| 요소 | 단가 | 설명 |
|------|------|------|
| **Base Fee** | $1/월 | 배포를 유지하는 기본 비용 |
| **Compute Units** | $3.20 / 백만 CU | 서버가 일한 만큼 |
| **Requests** | $1.20 / 백만 건 | 요청 수만큼 |

### Compute Unit(CU)이란?

- **CPU 1초 사용 = 18 CU**
- **메모리 1GB를 1초 사용 = 2 CU**

### Cold Start

Autoscale 앱은 15분 무활동 후 **잠자기 모드**로 들어갑니다. 누군가 접속하면 2~5초 후 깨어납니다.

- **장점**: 안 쓸 때 요금 $0
- **단점**: 첫 번째 방문자가 2~5초 기다려야 함

### Max Instances 설정

트래픽이 몰리면 서버를 자동으로 여러 개 띄웁니다. **해커톤 앱이라면 1~2개면 충분**. 설정 안 하면 비용 폭탄 가능!

### 월 비용 공식

```
월 비용 = $1 (기본) 
        + (총 요청 수 ÷ 1,000,000) × $1.20
        + (총 Compute Units ÷ 1,000,000) × $3.20
```

### 앱 유형별 요청당 CU 참고

| 앱 유형 | 요청당 평균 CU | 참고 |
|---------|---------------|------|
| 간단한 페이지 렌더링 | 3~5 CU | HTML 반환만 |
| API 응답 (DB 조회) | 8~15 CU | 간단한 CRUD |
| AI API 호출 포함 | 20~50 CU | 외부 API 대기 시간 |
| 이미지 처리 | 50~200 CU | CPU 집약적 |

## 7. 경제성 판단 — 언제 다른 플랫폼을 고려할까?

### ✅ Replit이 유리한 경우
- 프로토타입 & MVP (인프라 설정 시간 = 0)
- 소규모 서비스 (MAU 5,000 이하)
- 트래픽 변동이 큰 서비스
- 비개발자/1인 개발자

### ❌ 다른 플랫폼이 유리한 경우
- MAU 5만 이상 (Compute Unit 비용 증가)
- 정적 사이트 대량 트래픽 (Vercel/Netlify)
- 복잡한 인프라 (AWS/GCP)

### MAU별 플랫폼 추천

| MAU | 추천 플랫폼 | 이유 |
|-----|-----------|------|
| 0~1,000 | **Replit** | 개발 속도가 핵심, 비용 거의 없음 |
| 1,000~5,000 | **Replit 또는 Railway** | 둘 다 합리적 |
| 5,000~50,000 | **Railway / Render** | 가격 대비 성능 |
| 50,000+ | **AWS / GCP + Vercel** | 인프라 최적화 필요 |

## 8. 비용 절감 팁 🛡️

### ① Max Instances 제한 (필수!)
- 해커톤/데모: **1개**
- 소규모 서비스: **2개**
- 무제한은 절대 금지!

### ② Budget(예산) 설정
- **Settings → Account → Billing**에서 Spending alerts 설정

### ③ Static으로 전환 가능한지 확인
백엔드 없는 사이트는 **Static 배포**로 비용을 거의 0으로 줄일 수 있습니다.

### ④ 무료 서비스 적극 활용

| 무료 서비스 | 무료 범위 | 용도 |
|------------|----------|------|
| Resend | 월 3,000건, 일 100건 | 이메일 알림 |
| Tavily | 월 1,000회 | 검색/리서치 |
| Unsplash/Pexels | 무제한 | 이미지 |

### ⑤ 크레딧 낭비 줄이기
- **한 번에 하나씩** 요청 → 실패 시 되돌리기 쉬움
- **Plan 모드 먼저** → 잘못된 방향으로 작업하는 것 방지
- **디자인 레퍼런스 먼저 전달** → UI 반복 수정 최소화 (가장 큰 비용 요인!)

### ⑥ 모니터링
- [replit.com/usage](https://replit.com/usage)에서 실시간 사용량 확인
- 배포 첫 주에 매일 한 번 체크

## 9. 핵심 요약

1. **백엔드 없으면 Static** → 거의 무료
2. **대부분의 해커톤 앱은 Autoscale** → 월 $1~$5
3. **항상 켜져야 하면 Reserved VM** → 월 $20부터
4. **Max Instances 설정은 필수** → 비용 폭탄 방지
5. **Core 플랜 $20/월이면** 개발 환경 + AI Agent + 배포 + DB + SSL 전부 포함
6. **Replit의 진짜 가치는 개발 속도** → 인프라 비용보다 시간이 더 비쌈

> 🎯 해커톤에서는 비용 걱정 말고 만들자. Free 플랜 + Autoscale이면 데모 기간 동안 $1~2면 충분합니다!
