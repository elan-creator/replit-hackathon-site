## 🔍 체크포인트당 비용 (실측 데이터)

"체크포인트"란 Agent에게 하나의 작업을 요청하는 단위입니다. 비용은 작업 복잡도에 따라 크게 달라집니다.

### 실제 스크린샷 기반 예시

| 항목 | 값 |
|------|-----|
| 작업 시간 | 4분 |
| 수행 액션 | 26개 |
| 읽은 코드 | 2,373줄 |
| 변경된 코드 | +512줄 / -25줄 |
| **Agent 사용 비용** | **$1.27** |

### 복잡도별 비용 범위

| 작업 유형 | 비용 범위 | 구체적 예시 |
|----------|----------|-----------|
| 🟢 **간단** | $0.3~0.5 | 텍스트 수정, 이미지 교체, 링크 변경 |
| 🟡 **중간** | $0.5~1.0 | 컴포넌트 레이아웃 조정, 스타일링, 반응형 |
| 🟠 **복잡** | $1.0~2.0 | 새 페이지 추가, API 연동, 기능 구현 |
| 🔴 **대규모** | $1.5~2.0+ | 전체 구조 분석 + 대규모 코드 생성 |

### 비용이 많이 드는 작업 vs 적게 드는 작업

| 비용 多 | 비용 少 |
|--------|--------|
| 디자인/레이아웃 반복 수정 (38%) | 텍스트 문구 수정 |

## 💡 비용 절감 팁

### 1. 종량제의 이점을 활용하라

- Autoscale 배포는 **방문자가 없으면 $0**
- 해커톤 홍보 기간에만 트래픽이 증가 → 비용도 그때만 증가
- 고정 서버를 유지하는 것보다 훨씬 경제적

### 2. 무료 서비스를 최대한 활용하라

| 무료 서비스 | 무료 범위 | 용도 |
|------------|----------|------|
| Resend | 월 3,000건, 일 100건 | 이메일 알림 |
| Tavily | 월 1,000회 | 검색/리서치 |
| Unsplash/Pexels | 무제한 | 이미지 |
| Pretendard 폰트 | 무제한 | 한글 서체 |
| React, Tailwind 등 | 무제한 | 오픈소스 |

### 3. 크레딧 낭비를 줄이는 방법

- **한 번에 하나씩** 요청 → 실패 시 되돌리기 쉬움
- **Plan 모드 먼저** → 잘못된 방향으로 작업하는 것 방지
- **디자인 레퍼런스 먼저 전달** → UI 반복 수정 최소화
- **에러 메시지 정확히 전달** → Agent가 빠르게 해결

### 4. Credit Pack 활용

Core 플랜의 기본 크레딧을 다 쓰면 **Credit Pack을 추가 구매**할 수 있습니다:
- 추가 크레딧 단위로 구매 가능
- 해커톤처럼 집중 개발 시기에 유용
- 구독 업그레이드보다 유연한 선택
